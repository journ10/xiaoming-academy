# 小明书院 Roguelite 学习游戏改造方案

> 文档定位：把当前“章节路线 + 多入口 RPG 面板”改造为“以一局一局刷题为核心”的备考 Roguelite 学习游戏。本文是实现前的主方案，后续代码、测试、UI 和剧情改造都以此为准。

---

## 1. 改造结论

当前游戏最大问题不是题库、分类或单个页面样式，而是**第一体验没有闭环**：

- 玩家进入后同时看到路线、建议、题阵、短课、综合模拟、错题复盘、章节提示、卷宗、知识图谱和底部导航。
- 玩家不知道“现在应该点哪里开始”。
- 剧情是独立入口，读完几句话只改变“已读”状态，不改变题目、规则、奖励或下一步。
- 章节像题目分类列表，不像游戏副本。
- 多个系统并列露出，导致像学习数据面板，而不像游戏。

改造方向：

> **首页只服务开局。玩家选择本局目标和流派，进入一局题阵，在一局内遭遇题目、风险、心魔和奖励，结束后得到报告和下一局建议。**

剧情不删除，但不再作为独立页面存在。剧情必须绑定三个位置：

1. **开局规则**：告诉玩家这局为什么这样打。
2. **遭遇变化**：心魔、精英题、章节机制出现时改变压力。
3. **结算反馈**：解释这局暴露出的学习问题，并给下一局目标。

如果剧情不改变规则、目标、奖励或学习判断，就不出现在主体验里。

## 2. 目标体验

### 2.1 一句话

玩家作为“小明书院巡游者”，每次开一局选择学习目标和流派，用 5-8 道真题穿过题阵，答对获得成长，答错生成心魔，结算后得到下一局最该做的学习建议。

### 2.2 玩家进入后的 10 秒

玩家第一屏只需要理解三件事：

1. 现在推荐我做什么。
2. 点哪个按钮开始。
3. 这一局大概会发生什么。

首屏主行动：

```text
今日推荐：心魔净化
原因：你在“学习心理与认知机制”连续出现概念混淆。

[开始一局]

本局目标：净化 3 个概念混淆心魔
推荐流派：复盘
预计时长：5 分钟
```

首屏不展示题库审计数字，不展示路线列表，不展示完整知识图谱，不展示全部系统入口。

### 2.3 单局体验

一局不是“做 5 道题”，而是一个小型冒险：

```text
选择目标 -> 选择流派 -> 题阵遭遇 1 -> 题阵遭遇 2 -> 精英/心魔遭遇 -> 终局题 -> 本局报告 -> 下一局建议
```

每道题都必须产生即时反馈：

- 是否答对。
- 正确答案和解析。
- 错因判断。
- 本局资源变化。
- 是否生成或净化心魔。
- 下一题风险变化。

### 2.4 长期体验

玩家不是“通关七章”，而是持续做三类局：

- **探索局**：开新题，扩大知识版图。
- **净化局**：处理错题心魔，降低重复错误。
- **冲刺局**：混合知识域，训练考试迁移。

长期成长来自：

- 知识域掌握度提升。
- 流派等级提升。
- 心魔数量下降。
- 错题画像变清晰。
- 推荐策略越来越准。

## 3. 改造前后对比

| 模块 | 当前问题 | 改造后 |
|---|---|---|
| 首页 | 地图、建议、按钮、卷宗、导航同时出现 | 只显示推荐局、开始按钮、少量次级入口 |
| 主线 | 章节像分类列表 | 知识域变成本局题池和机制来源 |
| 剧情 | 独立读几句话，读完无变化 | 绑定开局、遭遇、结算，必须改变规则或目标 |
| 短课 | 和战斗并列，入口感弱 | 成为局内“补给/提示/复盘”动作 |
| 战斗 | 是一个场景入口 | 成为主要游戏本体 |
| 心魔 | 错题列表 | 下一局目标和局内敌人 |
| 仪表 | 常驻信息太重 | 变成结算后和二级入口查看 |
| 队伍/羁绊 | 与核心刷题关系弱 | 首版隐藏或删除 |
| 存档/设置 | 顶部操作可以保留 | 收进次级操作区，降低视觉优先级 |

## 4. 核心循环设计

### 4.1 5 秒循环：每题反馈

玩家行为：

1. 看题干和机制提示。
2. 选择招式。
3. 选择答案。
4. 释放破招。
5. 看即时结算。

反馈内容：

- 答对：伤害、奖励、连破、知识点点亮。
- 答错：正解、解析、错因、心魔生成、心力变化。
- 观照：给提示但降低奖励。
- 强攻：提高收益但加重失败代价。

学习价值：

- 答题不是黑箱。
- 错题不是失败，而是生成可处理的学习对象。
- 玩家每题都知道自己为什么错、下一步怎么补。

### 4.2 3-8 分钟循环：一局题阵

首版建议每局 **5 题**，沿用当前实现基础，降低重构风险。后续可以扩展到 7-8 道遭遇。

标准局结构：

| 位置 | 类型 | 作用 |
|---|---|---|
| 1 | 热身题 | 低压力进入本局题池 |
| 2 | 常规题 | 验证本局流派收益 |
| 3 | 机制题 | 引入知识域特殊规则 |
| 4 | 风险题或心魔题 | 触发错因诊断、净化或惩罚 |
| 5 | 终局题 | 结算本局目标 |

本局成功条件按模式变化：

- 探索局：答对至少 3 题，且至少点亮 1 个新概念。
- 净化局：至少净化 1 个心魔。
- 冲刺局：完成全部题目，并输出跨域薄弱点。

失败不是清零，而是进入“学习债务”：

- 心力归零，局失败。
- 已答题仍记录。
- 错题转化为心魔或强化已有心魔。
- 结算给出下一局建议，不让玩家只看到“失败”。

### 4.3 15 分钟循环：三局组合

系统推荐 15 分钟学习节奏：

1. 探索局：开新题。
2. 净化局：处理刚暴露的错题。
3. 冲刺局：混合检验。

连续三局后提示休息或导出复习清单，避免把备考游戏做成无脑刷题器。

## 5. 本局模式

### 5.1 探索局

目标：推进新知识。

题源：

- 优先当前推荐知识域。
- 只使用分类可靠、质量干净的主线题。
- 避开已反复答对的题。

局内特点：

- 奖励“新概念点亮”。
- 错题生成新心魔。
- 适合首次学习或日常推进。

首屏文案示例：

```text
探索局 · 法规政策营
目标：点亮 3 个法规关键词
风险：措辞陷阱会提高
推荐：稳修
```

### 5.2 净化局

目标：处理错题心魔。

题源：

- 玩家历史错题。
- 高压力心魔优先。
- 最近重复错的概念优先。

局内特点：

- 错题会以“心魔遭遇”形式出现。
- 答对降低压力，连续答对净化。
- 答错会强化心魔，但必须给出更明确的复习建议。

首屏文案示例：

```text
净化局 · 镜像心魔
目标：净化 2 个概念混淆心魔
风险：相似概念选项会更容易混淆
推荐：复盘
```

### 5.3 冲刺局

目标：模拟考试迁移。

题源：

- 多知识域混合。
- 优先已学但未稳定掌握的题。
- 排除人工待归类题和质量不稳题。

局内特点：

- 不强调剧情。
- 强调节奏、正确率、薄弱点报告。
- 结算给“下次冲刺前先补什么”。

首屏文案示例：

```text
冲刺局 · 混合秘卷
目标：完成 5 题跨域检验
风险：每题机制不同
推荐：稳修
```

## 6. 流派与招式

### 6.1 流派定位

流派是**开局前的构筑选择**，影响整局。它不是纯数值皮肤，而是学习策略。

首版保留 3 个基础流派，避免选择过载。

| 流派 | 游戏效果 | 学习含义 | 适合 |
|---|---|---|---|
| 稳修 | 心力 +1，奖励略低 | 稳扎稳打 | 新手、冲刺前保守练习 |
| 突击 | 连对奖励高，答错惩罚高 | 限时冲刺、保持专注 | 已熟悉题池 |
| 复盘 | 心魔净化收益高，新题收益低 | 查漏补缺 | 错题多时 |

后续再解锁领域流派：

- 律令：法规关键词收益提高。
- 观心：概念混淆题提示增强。
- 深读：短课收益提高。

### 6.2 招式定位

招式是**每题前的风险选择**，影响当前题。

| 招式 | 效果 | 玩家决策 |
|---|---|---|
| 稳破 | 正常收益、正常风险 | 不确定时的默认选项 |
| 强攻 | 奖励提高，答错惩罚提高 | 确定答案时赌收益 |
| 观照 | 显示提示，奖励降低 | 学习优先，不硬猜 |

流派 + 招式形成一局内的主要策略：

- 稳修 + 观照：安全学习。
- 突击 + 强攻：高风险高收益。
- 复盘 + 稳破：稳定净化错题。

## 7. 知识域机制

当前重新分类后的六个知识域是正确基础。改造后不再把它们做成“章节列表”，而是做成题池区域和机制来源。

| 知识域 | 游戏区域名 | 核心机制 | 学习训练 |
|---|---|---|---|
| 教育法律法规与政策制度 | 法规政策营 | 法条迷雾 | 关键词、权利义务、措辞精确 |
| 教育学原理、课程与教学 | 课程教学工坊 | 案例沙漏 | 原理应用、教学情境判断 |
| 学习心理与认知机制 | 学习心理实验室 | 概念镜像 | 相似概念区分、多选完整性 |
| 学生身心发展与个体差异 | 学生发展观察站 | 阶段刻度 | 年龄阶段、发展特征 |
| 德育、班级管理与家校协同 | 德育班管现场 | 策略连锁 | 情境处理、顺序决策 |
| 教师职业素养与专业规范 | 教师素养星台 | 职业天平 | 师德边界、规范判断 |

每局只突出一个主机制，避免首版一次暴露过多规则。

## 8. 剧情系统重构

### 8.1 删除独立剧情入口

底部导航和主界面不再出现“剧情”作为主入口。

原因：

- 独立剧情没有学习收益。
- 玩家会以为必须先看剧情才能玩。
- 读几句话不改变规则，造成无效点击。

### 8.2 剧情保留为事件

剧情改为“事件文本”，只在必要时出现。

| 位置 | 作用 | 示例 |
|---|---|---|
| 开局简报 | 改变本局目标或规则预期 | “这局镜像心魔较多，别急着强攻。” |
| 心魔出场 | 强化错题不是失败，而是学习对象 | “它不是敌人，是你还没分清的概念。” |
| 精英题 | 提醒特殊机制 | “这题有措辞陷阱，先看限定词。” |
| 结算 | 给学习解释和下一局建议 | “你不是不会法规，是容易漏看‘应当’。” |
| 里程碑 | 表达长期成长 | “你连续净化 10 个心魔，复盘流派解锁。” |

### 8.3 角色职责

角色不再作为剧情页里的说话人，而是作为反馈系统的人格化界面。

| 角色 | 新职责 |
|---|---|
| 明澈 | 规则、精确性、通关条件说明 |
| 阿芷 | 错因解释、复盘建议、安抚失败 |
| 青岚 | 风险收益、连破、突击局反馈 |
| 小墨 | 题阵状态、秘卷变化、长期里程碑 |

### 8.4 剧情出现原则

任何剧情文本必须通过以下检查：

1. 它是否改变玩家接下来要做什么？
2. 它是否解释本局规则或风险？
3. 它是否帮助玩家理解自己的错因？
4. 它是否奖励一个明确里程碑？

四项都不满足则删除。

## 9. UI 改造

### 9.1 全局信息架构

新结构：

```text
HUD：品牌 + 4 个本局状态 + 设置/存档
主舞台：当前唯一主任务
侧栏：可折叠，默认只显示本局目标或结算摘要
底部导航：首版移除
```

持久 HUD 只显示当前答题局状态：

- 心力
- 本局进度
- 当前目标
- 题眼状态

不显示：

- 能量
- 星辉
- 书页
- 星砂
- 灵签
- 题库总量
- 待归类数量
- 题位
- OCR 复核
- 内置题库状态

这些属于维护信息，只能出现在文档或开发审计文件中。

### 9.2 首页：开局台

首页结构：

```text
小明书院
今日推荐：心魔净化

本局目标
净化“概念混淆”心魔 2 个

推荐流派
复盘

[开始一局]

次级入口：
[换目标] [学习报告] [心魔回廊] [存档]
```

首页最多一个主按钮。

### 9.3 目标选择页

只有当玩家点击“换目标”时出现。

```text
选择本局目标

[探索新题]
推进推荐知识域，点亮新概念。

[净化心魔]
处理错题，降低重复错误。

[综合冲刺]
混合题池，检验考试迁移。
```

### 9.4 流派选择页

在进入一局前出现，首版最多 3 个流派。

每个流派必须显示：

- 收益
- 风险
- 为什么推荐或不推荐

### 9.5 局内页面

局内页面优先级：

1. 当前第几题。
2. 本题机制。
3. 题干。
4. 答案选项。
5. 招式选择。
6. 提交按钮。
7. 本局目标进度。

局内不显示完整知识图谱、章节列表或长卷宗。

### 9.6 结算页

结算页是学习价值最高的页面，必须清晰：

```text
本局报告

结果：成功 / 未完成
答对：4 / 5
新概念：2
净化心魔：1
新增心魔：1

主要错因：概念混淆
解释：你在“强化/惩罚”类概念上容易把行为结果和刺激变化混在一起。

下一局建议：
[继续净化] [去短课补概念] [综合冲刺检验]
```

结算页必须给出下一步，不让玩家回到复杂地图里自己找。

## 10. 题库与学习规则

### 10.1 题目池规则

运行时内部继续使用重新分类后的题库审计结果，但不把审计口径展示给玩家。

首版规则：

- 主线探索、净化、冲刺只使用分类可靠且质量干净的题。
- 待人工归类题不进入玩家题阵。
- 暂缓或低质量题不进入正式局。
- 这些过滤规则只在代码和文档中可见，不在游戏 UI 中出现。

### 10.2 短课规则

短课不再是首页主入口，而是三种触发方式：

1. 玩家在题内选择“观照”时展示关键提示。
2. 答错后结算建议进入短课补漏。
3. 净化局中心魔要求先复看题眼。

短课是局内或局后补给，不再和“开始一局”争夺主入口。

### 10.3 错题心魔规则

答错后必须创建或更新心魔：

- 记录题目。
- 记录错因。
- 记录压力值。
- 记录最近一次解释。
- 进入后续净化局候选池。

心魔净化条件：

- 同题或同概念连续正确。
- 或先完成短课，再答对心魔题。

净化后必须反馈：

- 心魔压力下降。
- 相关概念掌握度提升。
- 下次推荐是否改变。

## 11. 系统删改清单

### 11.1 保留并强化

- 真实题库加载。
- 六域重新分类。
- 题目质量过滤。
- 答题战斗。
- 招式选择。
- 流派/学习风格。
- 错题心魔。
- 结算报告。
- 复习清单导出。
- 存档导入导出。

### 11.2 改造

| 当前系统 | 改造方式 |
---|---|
| 世界地图 | 改为开局台 |
| 章节路线 | 改为题池区域和机制来源 |
| 剧情页 | 删除独立入口，改为事件文本 |
| 右侧卷宗 | 默认收起或改为本局目标摘要 |
| 底部导航 | 首版移除，必要入口放在结算或次级菜单 |
| 短课练功 | 改为局内/局后补给动作 |
| 综合模拟 | 改为冲刺局 |
| 错题复盘 | 改为净化局 |

### 11.3 首版隐藏或删除

- 队伍入口。
- 羁绊剧情。
- 独立章节提示按钮。
- 完整路线列表首屏展示。
- 数据审计类 HUD。
- 与本局无关的长期面板。

这些不是永远不能做，而是不能和核心循环抢首屏。

## 12. 实现阶段

### 阶段 1：运行模型重构

目标：建立 Roguelite 单局数据结构。

需要新增或调整：

- `runMode`: `explore | purify | sprint`
- `runBuild`: 当前流派
- `runGoal`: 本局目标
- `encounters`: 本局题目与遭遇列表
- `runResult`: 本局结算
- `nextRecommendation`: 下一局建议

验收：

- 可以从题库生成探索局、净化局、冲刺局。
- 每局有明确目标。
- 结算能输出答对数、心魔变化、推荐下一局。

### 阶段 2：首页改为开局台

目标：玩家打开游戏只看到一个主行动。

改造：

- `world` 场景改为开局台。
- 删除首屏章节路线列表。
- 删除首屏五个并列命令按钮。
- 主按钮为“开始一局”。
- 次级入口降级为文本按钮或折叠菜单。

验收：

- 新玩家 10 秒内能知道从哪里开始。
- 首屏没有“剧情”主入口。
- 首屏没有题库审计信息。

### 阶段 3：目标与流派选择

目标：让开局有构筑决策。

改造：

- 新增目标选择界面。
- 新增流派选择界面。
- 推荐目标和推荐流派由玩家状态生成。
- 默认可以一键使用推荐配置开始。

验收：

- 玩家可以一键开局。
- 玩家也可以手动换目标和流派。
- 每个选择都展示收益与风险。

### 阶段 4：局内题阵体验

目标：把“战斗”改成主游戏本体。

改造：

- 局内显示本局目标进度。
- 题目按遭遇顺序推进。
- 题目机制由知识域和模式决定。
- 答错后展示错因和心魔变化。
- 答对后展示奖励和掌握变化。

验收：

- 一局能从第 1 题打到结算。
- 每题反馈不只显示对错。
- 错题能影响后续净化推荐。

### 阶段 5：剧情事件化

目标：删除无效剧情页，让剧情服务局内变化。

改造：

- 移除独立剧情导航。
- 开局简报使用角色文本。
- 心魔出场使用角色文本。
- 结算建议使用角色文本。
- 里程碑才出现较长文本。

验收：

- 没有“点进去只看几句话”的剧情入口。
- 任意剧情文本都对应规则、目标、奖励或学习解释。

### 阶段 6：报告与推荐系统

目标：每局结束都告诉玩家下一步。

改造：

- 结算页显示本局表现。
- 输出主要错因。
- 推荐下一局模式。
- 推荐是否短课补漏。
- 支持导出复习清单。

验收：

- 玩家不用回地图找下一步。
- 错题多时推荐净化。
- 新题停滞时推荐探索。
- 多域不稳时推荐冲刺。

### 阶段 7：视觉收敛与移动端

目标：界面像游戏，不像后台。

改造：

- 常驻 UI 降噪。
- 移动端优先保证“开始一局”和“当前题”。
- 侧栏默认只显示本局信息。
- 长报告放到结算页，不在局内常驻。

验收：

- 桌面和移动端都不遮挡主内容。
- 首屏视觉焦点只有一个主行动。
- 按钮文案都是玩家行为，不是系统术语。

## 14. 验收标准

### 14.1 体验验收

- 首屏 10 秒内能判断应该点击“开始一局”。
- 不看说明也能完成第一局。
- 第一局结束后知道下一步该做什么。
- 剧情不再是无效阅读。
- 错题能被看见、解释、转化为下一局目标。

### 14.2 学习验收

- 每道错题都有解释或题眼。
- 答错后能形成错因记录。
- 心魔净化需要再次正确，而不是简单清除。
- 结算报告能指出主要薄弱点。
- 推荐系统能在探索、净化、冲刺之间切换。

### 14.3 UI 验收

- HUD 只显示玩家资源。
- 首屏不显示题库审计数字。
- 无底部多入口争夺焦点。
- 移动端主按钮和题干不被遮挡。
- 玩家行为按钮使用游戏语义，例如“开始一局”“进入题阵”“净化心魔”，不使用内部数据术语。

### 14.4 技术验收

- `npm test` 全部通过。
- 新增核心逻辑有单元测试。
- 玩家文案测试禁止审计词回流。
- 题库过滤测试保证待人工归类题不进正式局。
- 现有存档需要兼容或提供迁移默认值。

## 16. 风险与对策

### 风险 1：Roguelite 变成“换皮刷题”

对策：

- 每局必须有目标。
- 每题必须有风险选择。
- 结算必须影响下一局推荐。
- 错题必须进入心魔系统。

### 风险 2：选择过多，仍然乱

对策：

- 首屏只有一个主按钮。
- 目标选择默认推荐。
- 首版只有 3 个流派。
- 高级系统后置。

### 风险 3：剧情再次变成废话

对策：

- 删除独立剧情页。
- 剧情只绑定规则、目标、奖励、错因或里程碑。
- 测试或代码结构上不再提供“章节提示”主按钮。

### 风险 4：学习质量被娱乐性稀释

对策：

- 正确答案和解析必须显示。
- 错因诊断必须保留。
- 心魔净化必须靠再次答对。
- 待人工归类题不进入正式局。
- 结算报告优先学习建议，而不是只给奖励。

### 风险 5：一次改太大

对策：

- 分阶段实现。
- 第一阶段只改运行模型和首页。
- 先让“第一局”完整可玩，再迁移其他系统。
- 每阶段都有测试。

## 17. 最终产品形态

改造完成后，玩家不会再面对一张复杂地图和一堆入口。

玩家打开游戏看到的是：

```text
今日推荐：净化“概念混淆”心魔

你最近在学习心理题中连续混淆相似概念。
建议用“复盘”流派打一局净化题阵。

[开始一局]
```

玩家开始一局后，游戏带着他完成：

```text
选目标 -> 选流派 -> 答题遭遇 -> 错因反馈 -> 心魔变化 -> 本局报告 -> 下一局建议
```

这才是这个备考游戏应该有的核心：

- 必须会题。
- 错题能变成可处理对象。
- 每局有策略。
- 每天能玩几分钟。
- 剧情服务学习压力和反馈。
- 界面始终告诉玩家下一步。
