# 项目

- 游戏名：小明书院：秘卷巡游

## 0. 假设清单

- 当前版本可运行，但主要乐趣来自答题正确本身，地图、心魔、材料和血条没有改变玩家决策，因此仍像题库表单的视觉包装。
- 目标规模锁定为 2 周 Web 原型：1 个核心循环、最多 2 个次级循环、最多 6 个主要界面、最多 2 类内容家族、1 条简单成长线。
- 目标玩家是在碎片时间刷教招真题或答案解析的人，既想有效学习，也需要比普通题库更强的短局反馈。
- 题目 PDF 当前是扫描版答案解析页，完整题干与选项不能稳定自动得到；首版必须允许样例题和人工清洗 JSON 题库共存。
- 允许破坏性重构：当前的材料、建筑、地图、战斗表现都可以被替换，只保留“静态 Web、JSON 导入、本地存档、真题内容可接入”四个约束。
- 核心情绪目标不是“养成很多数值”，而是“我在压力下识破题眼并反杀错误惯性”。
- 学习/练功与战斗是并列模块，不强制先后顺序；练功提供讲解、题眼和心法收益，战斗用于检验学习成果。
- 内容生产顺序调整为先完成可玩的练功与战斗本体，再处理 PDF OCR 和题库导入扩展。
- 美术资源不能只做占位；首版需要成套 SVG 资源覆盖练功、破招式、路线节点、心魔状态、资源和战报。
- 2026-06-15 主线修订：游戏必须加入一次性序章说明、四名可识别角色、六章题库主线和玩家成长线。
- 正反馈来源被收紧：星辉、成长经验、羁绊和章节通关只能来自首次学习、答对题目或净化错题心魔。
- 通关语义被收紧：全章节通关应代表当前题库已经完成学习覆盖、正确检验和错题清理，而不是只走完剧情。
- 美术方向转为柔和书院奇幻，强调玫瑰粉、暖金、玉色、淡薰衣草、角色立绘感和章节封印感。
- 2026-06-15 二次重构：废弃旧三栏题库布局，改为全屏网页 RPG 舞台；主界面必须由世界地图、角色立绘、剧情对话、任务板和底部导航构成。
- 美术资源改用生成位图：世界地图、战斗背景、四名角色头像和四名角色立绘进入 `assets/generated/`，SVG 只保留为辅助符号。

## 0.5 方向锁定结果

- 入围方向数量：3
- 选中方向名称：秘卷巡游网页 RPG
- 选中理由：该方向把题库包装成可巡游的 RPG 世界：先看章节剧情，再用短课练功理解题眼，最后进入战斗检验。它比三栏题库包装更能形成地图探索、角色对话、学习准备、战斗检验、错题净化和章节通关的闭环。
- 舍弃方向摘要：书院经营养成被舍弃，因为核心快感落在局外升级，答题仍然平；卡牌构筑刷题被舍弃，因为它需要牌库、抽牌、平衡和大量 UI，首版范围过大。

## 1. 游戏一句话

- 玩家是谁：玩家是小明书院新入门的秘卷巡游者，与明澈、阿芷、青岚和小墨一起修复真题秘卷。
- 反复做什么：玩家在世界地图选择章节，看人物对话，完成短课练功，再进入题阵战斗，用答对和错题净化推动章节封印解除。
- 得到什么快感：玩家不是被动刷题，而是在“巡游地图、推进剧情、学会题眼、战斗检验、净化错题、点亮章节”的过程中获得可见成长。

## 2. 玩家承诺

- 身份幻想：我不是在填选项，我是在书院练功、理解题眼、与同伴一起修复被心魔污染的真题秘卷。
- 情绪承诺：每题都有学习与检验的正反馈，每章都有清晰封印目标，错题不是失败垃圾，而是可被追踪、削弱、净化的敌人。
- 差异化卖点：把教招真题刷题转成“讲解练功 + 风险招式 + 错题心魔 + 章节覆盖”的轻 RPG，而不是答完后才展示经验、金币和血条。

## 3. 核心玩法

- 核心动作：练功阅读讲解并抓题眼，选择题阵节点与破招式，选择答案，查看破招结果，决定继续路线或进入心魔回廊。
- 单轮流程：
  1. 首次运行显示序章说明，解释练功、战斗、心魔和章节通关目标。
  2. 章节地图展示六个主题秘境，每章显示学习数、答对数、心魔数和心法门槛。
  3. 路线节点展示题型、主题、风险修正和奖励修正。
  4. 玩家可在练功卷阅读讲解、题眼和来源标注，也可直接进入战斗。
  5. 首次练功给星辉、成长经验和阿芷羁绊，重复练功只用于复看讲解。
  6. 进入题阵后选择破招式：稳破、强攻、观照。
  7. 稳破提供正常奖励和正常惩罚。
  8. 强攻提高伤害与奖励，但答错时心力损失和心魔压迫翻倍。
  9. 观照给出题眼提示，但降低奖励并减少路线推进评价。
  10. 玩家选择答案后立即结算伤害、心力、灵页、心法熟练、星辉、羁绊和心魔状态。
  11. 每 5 题形成一次破阵报告，给出下一章目标和需要复训的心魔。
- 核心反馈：学习时题眼被摘入练功卷并提升同伴羁绊；正确时心魔被击破、章节推进、招式倍率生效、星辉和成长经验出现；错误时心魔留下具体诱因，进入回廊并带着压迫值回到后续局。
- 失败条件：心力归零、本局 5 题中累计 3 次强攻失误、或心魔压迫达到本局上限。
- 成功条件：完成 5 题节点并保持心力大于 0；章节成功为本章题目全部练过、答对过、无活跃心魔且主题心法达到门槛；高阶成功为全章节通关。

## 4. 节奏设计

- 前 2 分钟：玩家直接看到一条 5 节点路线，第一题立即要求选择破招式。第一题正确时立刻展示“强攻/稳破/观照”的差异，让玩家理解自己不是只在点答案。
- 前 10 分钟：玩家完成 2 局，每局 5 题。第一局偏安全，第二局出现路线修正：精英节点奖励更高但心魔压迫更强，回复节点降低奖励但恢复心力。
- 中段：玩家开始围绕主题短板选择路线。心法熟练不再只是百分比，而是改变招式手感，如稳破减少心力损失，强攻提高连击奖励，观照提升提示质量。
- 后段：玩家追求“无观照通关”“强攻连破”“心魔净化”等局内目标，而不是只堆总经验。
- 主线后段：玩家追求六章封印全部点亮；全章通关代表当前题库完成学习覆盖、正确检验和错题清理。
- 疲劳风险：如果题目前的破招选择没有真实差异，会退回普通题库；如果提示太强，会破坏学习；如果路线节点太多，会拖慢刷题节奏。

## 5. 世界观与叙事流程

- 世界一句话：历年真题答案解析被书院炉火炼成题阵，错误惯性会化作心魔反复诱导考生。
- 玩家身份一句话：玩家是负责进入题阵、识破题眼、净化错误惯性的破阵弟子。
- 核心冲突一句话：心魔利用玩家的熟悉错觉、赶题冲动和关键词误读来制造失误，玩家必须用心法控制风险。
- 主线人物：明澈负责序章、战报和章节门槛说明；阿芷负责练功陪伴和题眼摘记；青岚负责破招战斗和连破反馈；小墨负责秘卷封印、心魔回廊和错题净化。

### 5 幕结构
| 幕 | 叙事目标 | 新增压力或目标 | 玩法变化 |
|---|---|---|---|
| 第一幕 | 入门破阵 | 证明玩家能看出题眼并选择破招式 | 只开放稳破和观照，路线固定为 5 题 |
| 第二幕 | 强攻试炼 | 玩家被奖励倍率诱导，学习承担失误代价 | 开放强攻，错误产生心魔压迫 |
| 第三幕 | 心魔回廊 | 玩家开始把错题当作敌人处理 | 旧错题带压迫值回归，连续正确才能净化 |
| 第四幕 | 六脉分修 | 玩家主动选择补短板路线 | 主题心法影响招式参数 |
| 第五幕 | 真题秘卷 | 玩家挑战混合题阵并管理整局资源 | 路线节点带精英、回复、宝箱和压迫修正 |
| 第六幕 | 秘卷归元 | 玩家完成全部题库覆盖并清理心魔 | 六章全清判定解锁终局通关反馈 |

- 每幕新增压力或目标：每幕都改变玩家优化目标，从“答对”扩展到“选择风险”“处理旧错”“补短板”“管理路线”。
- 叙事可保留为文本的部分：书院来历、题阵解释、心法命名、章节标题。
- 必须由玩法承载的叙事部分：心魔诱导、心力压力、题眼识破、错题净化和强攻代价。

## 7. 规模与实现边界

- 必做：一次性序章、六章主线、四名角色、5 题一局、练功讲解、三种破招式、四类路线节点、心力失败、灵页奖励、星辉成长、角色羁绊、主题心法、心魔净化、局末报告、成套 SVG 美术、JSON 导入兼容入口。
- 可后做：更强动画、音效、更多心法被动、每日挑战、完整 OCR 管线、账号同步、排行榜。
- 首版砍掉：多材料背包、建筑升级、复杂装备、抽卡、牌组构筑、开放式地图、实时战斗、剧情分支、多人比较。
- 美术资源方案：继续使用现有书院背景图；节点、招式、心魔、心法、资源、战报、角色头像、章节封印和练功卷全部用统一 SVG 符号和 CSS 状态表现。
- 程序复杂点：破招式倍率必须被测试覆盖；路线节点不能只是装饰；心魔状态要和题目 ID 稳定绑定；旧存档需要迁移或重置；局末报告要能复盘每题事件。

### 原型验收

| 项目 | 目标 |
|---|---|
| 10 秒正反馈 | 玩家在第一题内选择破招式并看到不同倍率或提示代价 |
| 单局时长目标 | 5 题 2 到 4 分钟完成 |
| 核心循环闭环 | 路线选择 -> 练功或直接战斗 -> 破招式 -> 答题 -> 结算 -> 报告 -> 下一路线 |
| 首次说明 | 第一次运行出现角色对话序章，完成后不再自动打断，可手动重开 |
| 失败/成功状态明确 | 心力归零失败；完成 5 节点成功；报告指出下一局目标 |
| 通关含义 | 全章节通关要求当前题库全部完成章节剧情、练功、答对、无活跃心魔和章节心法门槛 |

## 9. 风险评审

| 风险类型 | 风险描述 | 触发阈值 | 应对动作 |
|---|---|---|---|
| 乐趣风险 | 破招式如果只是奖励倍率，会变成表层按钮 | 玩家 3 局内固定只选一种招式 | 调整节点和题型，让强攻、观照、稳破在不同场景有真实价值 |
| 节奏风险 | 观照提示会打断答题速度 | 每题平均超过 45 秒 | 提示限制为一行题眼或排除一个选项，不弹出长解释 |
| 制作风险 | 路线节点、心魔、心法同时改造会扩大实现量 | 单文件 app.js 超过清晰维护边界 | 破坏性拆分为 run-engine、question-engine、ui-renderers |
| 理解门槛风险 | 玩家不理解破招式代价 | 首题错误后不知道为何心力损失不同 | 在按钮上直接展示倍率和代价，不依赖说明文字 |
| 题材错位风险 | 游戏压力过强会削弱学习信任 | 玩家为奖励选择冒险而忽略复盘 | 报告优先展示题眼与错因，奖励只做辅助动机 |
| 主线空转风险 | 章节只是剧情标签，没有约束题库掌握 | 玩家不学习也能点亮章节 | 章节通关必须检查学习覆盖、答对覆盖、心魔清理和心法门槛 |
| 成长滥发风险 | 点击、跳过或答错也给奖励，会破坏学习信任 | 错题后星辉或羁绊增长 | 成长公式只接受首次学习、答对和净化事件 |
