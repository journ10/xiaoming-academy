# 小明书院娱乐化重设计方案

## 一句话定位

把“刷历年真题”做成一款静态 Web 端轻量 RPG：玩家不是在做题表单，而是在书院世界里闯秘境、破心魔、收集法器、修炼知识心法。

## 核心判断

上一版方向正确，但形态太像学习工具。新版要把题目包装成“行动”，把解析包装成“战斗反馈”，把错题包装成“可反复挑战的敌人”，把知识点掌握度包装成“修炼成长”。

目标不是做复杂游戏系统，而是让用户每答一题都有游戏反馈：

- 答对：释放招式、获得经验、掉落材料、推进秘境。
- 答错：生成错题心魔，进入错题斋，后续可驯化。
- 连胜：触发暴击、加成、额外掉落。
- 章节完成：解锁新地图、新建筑、新称号。

## 推荐主题

项目名仍可叫“小明书院”，游戏内副标题改为：

**小明书院：真题秘境**

世界观：

考生进入一座会把历年真题显化成秘境的书院。每个年份是一卷秘境，每个知识点是一条灵脉，每道错题会变成“心魔”。玩家通过答题修炼，把真题解析转化为心法。

这个主题的好处：

- 和教师考编题库天然贴合，不突兀。
- 能解释错题、复训、知识点掌握度等学习功能。
- 视觉上可以做成国风奇幻书院，比普通题库 UI 更有记忆点。
- 仍然适合静态 Web，不需要实时战斗或复杂服务器。

## 娱乐性设计

### 1. 真题秘境

把题库按年份、题型、知识点拆成关卡地图。

示例：

- 2013 卷：入门秘境
- 2018 卷：师德试炼
- 2021 卷：班级管理迷阵
- 错题卷：心魔回廊

每个秘境不是列表，而是一条路线：普通题、精英题、解析宝箱、最终模拟试炼。

MVP 可以先用 6-10 个节点假装地图，点击节点进入题目。

### 2. 答题战斗

答题界面改为“回合战”表达：

- 题干是敌人的招式描述。
- 选项是玩家的四个应对招式。
- 答对时敌方血量下降，出现“破招成功”。
- 答错时玩家损失心力，题目进入错题心魔。
- 解析不再叫“解析”，改成“破招笔记”。

注意：战斗只是表现层，不需要复杂数值。核心还是选择题判断。

### 3. 心法与法器

知识点掌握度升级为“心法等级”：

- 教育法规：律令心法
- 教育心理学：观心心法
- 教学设计：授业心法
- 教师职业道德：师德心法
- 班级管理：统御心法
- 儿童发展：童心心法

答题掉落材料：

- 书页
- 墨玉
- 星砂
- 灵签

材料用于升级建筑或法器。MVP 中先做展示和简单加成，不做复杂背包。

### 4. 错题心魔

错题不只是列表，而是“心魔收容”：

- 第一次答错：心魔出现。
- 第二次答对：心魔削弱。
- 连续两次答对：心魔净化，移出错题斋。

这比“错题本”更有目标感，也天然鼓励复习。

### 5. 每日秘境

每天随机 10 题，形成短局体验：

- 3 连胜：获得临时增益。
- 5 连胜：掉落稀有材料。
- 通关：给称号或书院声望。

私人项目可以先不做日期服务端校验，用本地日期和 localStorage 即可。

## 视觉方向

整体风格：**明亮国风奇幻 + 2.5D 游戏 UI + 高对比信息面板**。

避免：

- 米色纸张风全屏铺开，会像传统课件。
- 纯蓝白工具界面，会像后台系统。
- 过度深色，会削弱学习内容可读性。

推荐色彩：

- 主色：钴蓝、青绿
- 奖励色：琥珀金
- 危险色：朱红
- 背景：浅天青、雾蓝、夜幕蓝局部点缀

- 后续可继续生成建筑、角色、敌人、法器图标。

## 页面结构

### 首屏：秘境地图

用户打开后直接看到地图，不看说明页。

核心区域：

- 背景：书院秘境插画。
- 中心：关卡路线。
- 左侧：角色状态、心力、等级。
- 右侧：今日任务、错题心魔、掉落预览。

### 答题页：破招战斗

核心区域：

- 上方：敌人/心魔卡面 + 血量。
- 中间：题干。
- 下方：四个选项按钮。
- 右侧或底部：当前心法、连胜、奖励。

提交后：

- 答对：播放短动画，显示掉落。
- 答错：显示心魔提示和破招笔记。

### 结算页：修行报告

一局 5-10 题后展示：

- 击破心魔数量
- 掌握度提升
- 掉落材料
- 新解锁建筑或称号
- 推荐复训知识点

## 数据结构调整

题库仍然保持 JSON，但建议增加游戏字段：

```json
{
  "id": "edu-law-2013-01",
  "year": "2013",
  "type": "单项选择",
  "topic": "教育法规",
  "difficulty": 2,
  "enemy": "律令心魔",
  "realm": "2013 入门秘境",
  "stem": "题干",
  "options": [],
  "answer": "B",
  "explanation": "解析",
  "xp": 16,
  "coins": 10,
  "drops": ["书页", "星砂"]
}
```

如果 OCR 阶段暂时无法标注 `enemy` 和 `realm`，可以由程序按年份和 topic 自动生成。

## MVP 重做范围

第一版重做不追求大而全，建议只做这些：

1. 地图首页：6 个关卡节点，点击进入答题。
2. 战斗式答题：敌人血量、破招反馈、奖励掉落。
3. 错题心魔：错题从列表升级成可净化目标。
4. 心法面板：把掌握度改成六类心法等级。
5. 结算页：每 5 题给一次修行报告。
6. 视觉资产：至少生成一张背景图、一个主角、一个心魔、三类材料图标。

仍然保持：

- 静态 Web。
- localStorage 存档。
- JSON 导入题库。
- 中国大陆访问无远程依赖。

## 实现建议

当前项目可以继续用原生 HTML/CSS/JS，不急着换 React。

原因：

- 私人项目部署最简单。
- 玩法还在探索，少依赖更快改。
- 静态文件直接上传国内 OSS/COS 即可。

等玩法确认后再考虑：

- Phaser：如果要做更强动画、地图、粒子和小游戏。
- React/Vite：如果题库管理、页面状态复杂起来。
- Supabase/自建后端：如果后续需要账号和多端同步。

## 下一步执行顺序

1. 保留当前迁移版本作为备份。
2. 重做首页为“秘境地图”。
3. 重做答题面板为“破招战斗”。
4. 增加一局 5 题的结算流程。
5. 生成并替换首批游戏视觉资产。
6. 再接真实 OCR 题库。

## 成功标准

新版完成后，打开页面第一眼应该像一个轻量游戏，而不是一个题库管理页。

具体判断：

- 用户能看到地图、角色、心魔、奖励。
- 答题后有即时视觉反馈。
- 错题复习有“净化目标”，不是普通列表。
- 5 分钟内能完成一局，并看到结算成长。
- 上传静态文件即可部署。
