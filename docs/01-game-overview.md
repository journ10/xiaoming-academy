# 小明书院：文字巡游 — 游戏设计总纲

> 文档定位：项目唯一的设计总纲。本游戏为纯文字驱动的 RPG，所有画面由 CSS 渲染文本和文本按钮构成，无图片、无动画、无音频、无后端。

---

## 1. 一句话定位

把"刷历年真题"做成一款纯文字 Web 端轻量 RPG。玩家不是在填表单，而是在文字构成的书院世界里闯秘境、破心魔、修炼心法。

---

## 2. 世界观

### 2.1 世界设定

小明书院是一座"显化书院"——知识会变成秘境，遗忘会凝成雾。历年真题被炼成题阵，错误惯性会化作心魔反复诱导考生。

六章封印对应六类教招真题主题：教育法规、教育心理学、教学设计、教师职业道德、班级管理、儿童发展。每章封印需通过"练功+战斗+净化"三步修复。

### 2.2 黑色墨迹（叙事线索）

每章通关后，秘卷边缘出现黑色墨迹文字，是创始人留下的试炼提醒：
- 第一章：「面对错题，不要逃避」
- 第二章：「承认无知，才能求知」
- 第三章：「教与学，相辅相成」
- 第四章：「师德为先，行为世范」
- 第五章：「班级管理，不是控制，而是引导」
- 第六章：「理解儿童，才能理解教育」

---

## 3. 玩家承诺

- **身份幻想**：我不是在填选项，我是在书院练功、理解题眼、与同伴一起修复被心魔污染的真题秘卷。
- **情绪承诺**：每题都有学习与检验的正反馈，每章都有清晰封印目标，错题不是失败垃圾，而是可被追踪、削弱、净化的敌人。
- **差异化卖点**：把教招真题刷题转成"讲解练功 + 风险招式 + 错题心魔 + 章节覆盖"的轻 RPG，而非普通答题表单。

---

## 4. 核心循环

```
世界地图 → 剧情对话 → 短课练功 → 题阵战斗 → 心魔净化 → 章节通关
```

### 4.1 单局流程（5题一局）

1. **世界地图**：选择章节，查看进度和推荐行动。
2. **剧情对话**：硬编码文字对话，角色轮流发言推动目标。
3. **短课练功**：阅读题眼、来源和记忆动作，获得首次练功奖励。
4. **题阵战斗**：选择破招式（稳破/强攻/观照），选择答案，即时文字结算。
5. **心魔净化**：答错的题进入心魔回廊，连续答对两次可净化。
6. **破阵报告**：每局结束展示正确率、成长、错因和下一目标。
7. **章节通关**：当本章全部题目练过、答对过、无活跃心魔、心法达到门槛时，封印点亮。

### 4.2 胜负条件

- **失败**：心力归零；本局累计3次强攻失误；心魔压迫达到12。
- **成功**：完成5题并保持心力大于0。
- **章节成功**：本章题目全部练过、答对过、无活跃心魔、主题心法达到门槛。
- **高阶成功**：全六章通关，代表当前题库完成学习覆盖、正确检验和错题清理。

## 5. 角色

| 角色 | 定位 | 玩法职责 |
|---|---|---|
| **明澈** | 书院导师 | 序章说明、战报点评、章节门槛解释 |
| **阿芷** | 练功同伴 | 学习讲解、题眼摘记、练功奖励 |
| **青岚** | 战斗同伴 | 破招鼓励、答对反馈、连破表现 |
| **小墨** | 秘卷灵 | 错题心魔、净化、章节封印 |

角色对话硬编码在 `core.js` 的 `chapterNarrativeBeats` 和 `dialogueLine` 函数中，以及 `app.js` 的 `getIntroDialogue` 中。无立绘、无头像、无表情系统。

## 7. 技术约束

| 约束 | 说明 |
|---|---|
| 静态 Web | 无后端、无构建工具、无框架依赖 |
| 存档 | localStorage (`xiaoming-academy-text-game-v1`) |
| 题库导入 | JSON 文件导入，兼容题目数组或 `{questions:[]}` 格式 |
| 素材 | 零图片资源，纯 CSS 渲染文字界面 |
| 访问 | 中国大陆可直接访问，无远程依赖 |
| 测试 | Node test runner，`core.test.mjs` + `runtime.test.mjs` |
