# 小明书院：秘卷巡游 — UI 设计规范

> 当前基准：界面必须让玩家一眼知道“现在开始哪一局”。旧地图、章节卡片、底部导航和审计类 HUD 不再是主界面规范。

## 1. 设计原则

1. **首屏只服务开局**：推荐局、目标、流派、开始按钮必须是第一视觉层级。
2. **审计信息后台化**：源题位、OCR 复核、待归类数量不作为游戏 HUD。
3. **短局优先**：局内界面围绕 5 题题阵，减少旁支入口。
4. **反馈明确**：答错后立即显示错因和心魔影响。
5. **下一步可见**：战报必须有下一局建议按钮。

## 2. 全局布局

```css
.rpg-shell {
  display: grid;
  grid-template-areas:
    "hud hud"
    "stage quest";
  grid-template-columns: minmax(0, 1fr) 340px;
  grid-template-rows: auto minmax(0, 1fr);
  gap: 14px;
  min-height: 100dvh;
}
```

桌面端：

```text
┌──────────────────────────────────────────────┬──────────────┐
│ HUD：品牌 + 本局状态 + 存档操作               │              │
├──────────────────────────────────────────────┤              │
│ stage：开局台 / 选择 / 题阵 / 战报            │ quest：进度   │
│                                              │ 和下一步      │
└──────────────────────────────────────────────┴──────────────┘
```

移动端：

```css
.rpg-shell {
  grid-template-areas:
    "hud"
    "stage"
    "quest";
  grid-template-columns: 1fr;
}
```

不使用固定底部导航。主要入口在开局台和战报里出现。

## 3. HUD

HUD 只显示本局答题状态和存档操作。

```text
小明书院
秘卷巡游 · 文字题阵

心力 6/6 | 本局 1/5 | 目标 探索新题 | 题眼 待进入
[导入存档] [导出存档] [重置]
```

HUD 不显示：

- 能量。
- 星辉。
- 书页。
- 正式题总数。
- 待归类数。
- 源题位。
- OCR 复核数。
- 七章进度。

## 4. 开局台

开局台是首屏主界面。

```text
今日推荐
净化局 · 复盘流派

本局目标
清理 5 道活跃心魔，优先处理概念混淆。

[开始一局]
[换目标] [选择模式] [学习报告] [心魔回廊]

下一步建议
你有 3 个同类错因正在复燃，先净化再开新题。
```

视觉规则：

- “开始一局”是唯一主按钮。
- “换目标”“学习报告”“心魔回廊”是次级按钮。
- 推荐理由最多两行。
- 不展示完整知识图谱。

## 5. 模式选择

```text
选择本局目标

[探索局]
点亮未见过的题，扩大覆盖面。

[净化局]
处理活跃心魔和重复错题。

[冲刺局]
混合多学习域，训练考试切换。
```

模式卡片必须展示：

- 名称。
- 一句目标。
- 当前是否推荐。
- 预计题数：5 题。

## 6. 流派选择

```text
选择学习流派

[稳修]  稳定推进，容错更高
[突击]  连破收益高，答错更疼
[复盘]  错题收益高，新题较慢
```

流派卡片必须展示收益和风险，不只写风味文案。

## 7. 局内题阵

```text
净化局 · 第 3 / 5 遭遇
目标：清理教育心理学概念混淆心魔

题干：……

A. …
B. …
C. …
D. …

破招式：[稳破] [强攻] [观照]
[释放破招]
```

局内信息层级：

1. 本局目标和遭遇进度。
2. 题干和选项。
3. 破招式。
4. 日志和反馈。

局内不出现章节路线、剧情按钮、完整卷宗或题库审计数字。

## 8. 答题反馈

答对：

```text
答对。该知识点稳定度 +1。
青岚：这题你没有急，限定词抓住了。
```

答错：

```text
答错。生成镜像心魔。
错因：概念混淆。
建议：下一局优先净化同类题。
```

反馈必须包含正确性、错因或收益、下一步提示。

## 9. 战报

```text
本局报告

净化局 · 复盘
正确率 4/5
净化心魔 2
新增心魔 1
主要错因：审题失误

下一局建议
继续净化：还有 3 道同类错因未处理。

[按建议继续] [换目标] [看心魔] [学习报告]
```

战报不让玩家回到复杂地图里自己找下一步。

## 10. 右侧卷宗

右侧卷宗展示当前学习状态，不承担题库审计任务。

```text
当前进度
本局模式：净化局
当前学习域：学习心理与认知机制
练习 32 / 1144
答对 25 / 1144
活跃心魔 3
心法 18%

下一步
优先净化概念混淆心魔。
```

允许展示：

- 当前模式。
- 当前学习域。
- 练习/答对/心魔/心法。
- 下一步建议。

不展示：

- 4680 源题位。
- 4374 当前可玩题。
- 114 待人工归类。
- OCR 复核数。

## 11. 响应式规则

| 元素 | 桌面 | 移动 |
|---|---|---|
| HUD | 横向资源卡 | 自动换行 |
| stage | 主列 | 第一列 |
| quest | 右侧常驻 | stage 后方 |
| 模式卡片 | 3 列 | 1 列 |
| 流派卡片 | 3 列 | 1 列 |
| 战报按钮 | 横向排列 | 自动换行 |

所有按钮文本必须在 320px 宽度内不溢出。

## 12. 禁止项

- 首屏章节地图。
- 独立剧情按钮。
- 底部导航作为主入口。
- HUD 展示审计指标。
- 右侧卷宗塞满知识图谱树。
- 卡片里嵌套卡片。
- 用大段说明文字解释怎么玩。
