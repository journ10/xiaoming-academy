# 小明书院：文字巡游 — UI 设计规范

> 文档定位：前端实现的唯一 UI 参考。本游戏为纯文字驱动，所有界面由 CSS 渲染文本和文本按钮构成，零图片、零动画、零音频。

---

## 1. 设计原则

1. **文字优先**：所有信息用文字传达，无图标、无插图、无装饰图
2. **结构清晰**：CSS Grid 布局，信息层级明确
3. **反馈即时**：按钮悬停变色、选中高亮、答对/答错边框变色
4. **响应式**：桌面端左右分栏，移动端单列堆叠
5. **低干扰配色**：暖纸色背景 + 深墨文字 + 青绿强调

---

## 2. 色彩系统

**CSS 变量定义：** `styles.css` 第1-15行

```css
:root {
  color-scheme: light;
  --text-bg: #f7f1e6;          /* 页面背景 */
  --text-paper: #fffaf2;       /* 面板背景 */
  --text-ink: #211b18;         /* 主文字 */
  --text-muted: #70665d;       /* 次要文字 */
  --text-line: #ddd1c2;        /* 边框/分隔线 */
  --text-strong-line: #b9aa98; /* 按钮边框 */
  --text-accent: #2d6f62;      /* 青绿：强调/正确/选中 */
  --text-danger: #9d3c45;      /* 朱红：错误/危险 */
  --text-focus: rgba(45, 111, 98, 0.22); /* 焦点轮廓 */
  --text-shadow: 0 12px 32px rgba(55, 43, 31, 0.08); /* 面板阴影 */
  --radius: 8px;
}
```

**语义色彩映射：**

| 状态 | 边框色 | 背景色 | 文字色 |
|---|---|---|---|
| 默认 | --text-strong-line | #fffdf8 | --text-ink |
| 悬停 | --text-accent | 不变 | 不变 |
| 选中 | --text-accent | #f2faf6 | --text-accent |
| 主按钮 | --text-accent | #edf8f4 | --text-accent |
| 正确 | --text-accent | 默认 | 默认 |
| 错误 | --text-danger | 默认 | 默认 |
| 禁用 | 灰 | 灰 | 灰 | opacity: 0.5 |

## 3. 全局布局

**Grid 结构：** `styles.css` 第86-99行

```css
.rpg-shell {
  display: grid;
  grid-template-areas:
    "hud hud"
    "stage quest"
    "nav nav";
  grid-template-columns: minmax(0, 1fr) 320px;
  grid-template-rows: auto minmax(0, 1fr) auto;
  gap: 14px;
  min-height: 100dvh;
  width: min(1180px, 100%);
  margin: 0 auto;
  padding: 16px;
}
```

**桌面端（≥761px）：**

```
┌──────────────────────────────┬─────────────┐
│ [HUD]                        │             │
├──────────────────────────────┼─────────────┤
│                              │  quest      │
│  stage（游戏主画面）          │  （当前卷宗）│
│                              │             │
│                              │             │
├──────────────────────────────┼─────────────┤
│ [底部导航] 地图 练功 战斗 心魔 队伍 日课 │             │
└──────────────────────────────┴─────────────┘
```

**移动端（≤760px）：** 单列堆叠，quest 下移，底部导航固定

```css
@media (max-width: 760px) {
  .rpg-shell {
    grid-template-areas: "hud" "stage" "quest" "nav";
    grid-template-columns: minmax(0, 1fr);
  }
}
```

## 4. 页面组件

### 4.1 HUD（顶部信息栏）

**布局：** 3列 Grid — 品牌区 | 资源区 | 操作区

**品牌区：**
- 标题："小明书院"（h1，1.35rem）
- 副标题："文字巡游"（p，muted 色）

**资源区（4个统计）：**
- 心力：`6/6`
- 能量：`12/12`
- 星辉：`0`
- 书页：`0`

每个统计：左侧竖线分隔 + 标签（小字 muted）+ 数值（1.05rem）

**操作区：**
- 来源标签：`内置题库` / 导入文件名
- 题目计数：`0 题 · 0 章`
- 按钮：导入、序章、重置（重置为 danger 样式）
- 隐藏文件输入框（`<input type="file" accept="application/json,.json" hidden>`）

### 4.2 游戏主画面（Stage）

所有场景共用 `textScreen` 组件结构：

```
┌─────────────────────────────┐
│  kicker（场景标签）            │
│  标题（当前内容标题）          │
│  引言（一句话说明）            │
│                              │
│  ┌─────────────────────────┐│
│  │ 面板1（标题 + 内容）      ││
│  └─────────────────────────┘│
│  ┌─────────────────────────┐│
│  │ 面板2（标题 + 内容）      ││
│  └─────────────────────────┘│
│                              │
│  [按钮1] [按钮2] [按钮3]      │
│  日志: 当前状态提示            │
└─────────────────────────────┘
```

**CSS：**
```css
.text-screen {
  display: grid;
  grid-template-rows: auto minmax(0, 1fr) auto;
  gap: 18px;
  min-height: 100%;
  padding: 24px;
}
```

### 4.3 世界地图（World）

**内容：**
- kicker: "地图"
- 标题: 当前章节标题（或"真题秘卷"）
- 引言: 章节摘要

**面板1 — 章节列表：**
每个章节一个按钮，显示：
```
第一章 律令花窗 · 行动标签
  练功 3/6 · 答对 2/6 · 心魔 1
```

**面板2 — 当前目标：**
- 推荐行动原因文字
- 进度条（章节完成度百分比）

**按钮：** 读剧情、练功、进入战斗、处理心魔

### 4.4 剧情对话（Story）

**内容：**
- kicker: "序章" / "剧情"
- 标题: 当前说话者名称（明澈/阿芷/青岚/小墨）
- 引言: 当前台词文本

**面板 — 已读记录：**
所有已读对话的纯文本列表：
```
明澈：小明书院是一座显化书院...
阿芷：先练功，把讲解拆成题眼...
```

**按钮：** 继续 / 完成剧情、去练功、去战斗

### 4.5 短课练功（Training）

**内容：**
- kicker: "练功"
- 标题: 讲解标题（如"教育法规 · 2013讲解"）
- 引言: 练功目标提示

**面板1 — 题眼：**
- 关键词（粗体）
- 完整解析（muted 色）

**面板2 — 当前阵位：**
- 节点名称 + 风味描述
- 奖励预览文字

**按钮：** 完成练功（主按钮）、进入战斗、回地图

**状态：** 首次练功有奖励，复看无奖励。

### 4.6 题阵战斗（Battle）

**内容：**
- kicker: "战斗"
- 标题: `心魔名称 · 节点名称`（如"律令心魔 · 常阵"）
- 引言: 题干文本

**面板1 — 选项：**
4个按钮，每个显示：
```
A. 选项文本
B. 选项文本
C. 选项文本
D. 选项文本
```

选中后边框变色，提交后：
- 正确答案：边框青绿
- 错误选择：边框朱红

**面板2 — 招式：**
3个按钮：稳破 Lv.1 / 强攻 Lv.1 / 观照 Lv.1
选中后高亮，显示招式描述小字。

**反馈面板（提交后）：**
- 成功：伤害值、星辉+、修为+
- 失误：正解答案、心力变化、学习检查提示

**按钮（未提交）：** 释放破招、回练功、回地图
**按钮（已提交）：** 下一题 / 查看战报、回看讲解、回地图

**日志：** 当前选择 + 当前招式

**键盘快捷键：** 按数字键1-4直接选择对应选项。

### 4.7 心魔回廊（Review）

**内容：**
- kicker: "心魔"
- 标题: 有错题则"错题回廊"，无则"暂无心魔"
- 引言: 说明文字

**面板 — 心魔列表：**
每个心魔一个卡片：
```
律令心魔
主题：教育法规 · 压力：3
```

**按钮：** 开始净化、去战斗制造检验、回地图

### 4.8 队伍与法器（Roster）

**内容：**
- kicker: "队伍"
- 标题: "同伴与法器"
- 引言: 总羁绊值

**面板1 — 同伴：**
4个卡片（2列Grid）：
```
明澈
书院导师
羁绊 0
```

**面板2 — 法器：**
按钮列表，每个显示：
```
碧灵笔 Lv.1
消耗 书页+8 · 星砂+2
```
可升级时主按钮样式，不可升级时普通样式。

**按钮：** 回地图、去日课

### 4.9 日课（Daily）

**内容：**
- kicker: "日课"
- 标题: "今日清单"
- 引言: 说明文字

**面板 — 任务：**
每个任务一个卡片：
```
晨课三问
完成练功短课，把讲解转成题眼记忆。
进度: 1/3
奖励: 书页+3 · 星砂+1
[进度条]
```

**按钮：** 按日课练功、按日课战斗、回地图

### 4.10 破阵报告（Report）

**内容：**
- kicker: "战报"
- 标题: 评价等级（"心力告急" / "破阵通明" / "心法成形" / "回炉练功"）
- 引言: 正确率、答题数、错题数

**面板1 — 本轮收益：**
- 星辉 +N
- 修为 +N
- 材料（如有）
- 下一步建议

**面板2 — 事件：**
逐题记录：
```
教育法规：答对，练功检验：已回看讲解...
教育心理学：答错，战斗检验：未练功...
```

**按钮：** 继续战斗、去心魔、回地图

## 5. 当前卷宗面板（Quest Panel）

右侧固定面板，始终显示当前章节进度：

```
当前卷宗
─────────────────
章节      第一章 律令花窗
剧情      完成 / 未读
练功      3/6
答对      2/6
心魔      1
律令心法  35/70
总进度    1/6
```

每项为 `quest-line`：标签 + 数值，分隔线分隔。

## 6. 底部导航

6个按钮，等分宽度：

```
[地图] [练功] [战斗] [心魔] [队伍] [日课]
```

当前场景按钮高亮（边框青绿、背景浅绿、文字深绿）。

点击切换场景：
- 练功/战斗：自动确保路线存在，否则重置
- 其他：直接切换

## 7. 文字排版

| 层级 | 字体 | 大小 | 用途 |
|---|---|---|---|
| 品牌标题 | ui-serif, Songti SC, Noto Serif CJK SC | 1.35rem | 书院名称 |
| 场景标题 | 同上 | clamp(1.35rem, 2vw, 1.9rem) | 页面标题 |
| 面板标题 | 同上 | 默认 | h3 |
| 正文 | 同上 | 默认 | 题干、解析、对话 |
| 按钮标签 | 同上 | 默认 | 按钮内文字 |
| 标签/辅助 | 同上 | 0.82rem | 统计标签、muted 文字 |
| 日志 | 同上 | 默认 | 底部状态提示 |

## 9. 通用组件规格

### textPanel
- 边框: 1px solid var(--text-line)
- 背景: var(--text-paper)
- 圆角: var(--radius)
- 内边距: 16px
- 选中态: 边框青绿、背景浅绿

### textChoice / textButton
- 最小高度: 42px
- 边框: 1px solid var(--text-strong-line)
- 背景: #fffdf8
- 圆角: var(--radius)
- 内边距: 12px 14px
- 文本左对齐
- 支持小字描述（small 标签）

### meter（进度条）
- 容器: 8px 高，圆角999px
- 背景: #f0e8dc
- 填充: var(--text-accent)
- 宽度由 CSS 变量 `--value` 控制

### toast
- 固定右下角
- 最大宽度: min(420px, calc(100vw - 32px))
- 2.2秒自动消失
- 透明度渐变

---

## 10. 无障碍

- 所有按钮有 `type="button"`
- 导航按钮有 `aria-label` 和 `aria-current`
- 主舞台有 `aria-live="polite"`
- toast 有 `role="status"` 和 `aria-live="polite"`
- skip-link 跳转到主舞台
- 焦点可见：3px outline，青绿色
