# 小明书院：文字版 UI 规范

## 1. 设计目标

界面服务真题阅读、选项作答、即时反馈和下一步行动。

优先级：

1. 题干和选项清楚。
2. 提交动作容易到达。
3. 反馈能立即解释对错。
4. 学习报告能直接引导下一局。

## 2. 全局信息结构

```text
顶部状态：小明书院 / 当前题阵 / 进度
主区域：开局台、题阵、答后反馈、学习报告
辅助区域：本局目标、心魔压力、下一步建议
```

移动端主区域优先。辅助信息必须排在主操作之后，不抢题干和选项空间。

## 3. 开局台

```text
小明书院
真题题阵

今日推荐
净魔题阵 · 复盘流派

推荐理由
你最近连续漏看题干限定词。

本局目标
净化 2 个审题失误心魔。

[进入题阵]

[换目标] [查看心魔] [学习报告]
```

规则：

- “进入题阵”是唯一主按钮。
- 推荐理由最多两行。
- 次级入口不超过三个。
- 不展示题库审计信息。

## 4. 目标与流派选择

目标选择用于换掉系统推荐。

```text
选择本局目标

[拓新题阵]
接触新题，扩大覆盖面。

[净魔题阵]
处理活跃心魔和重复错因。

[冲刺题阵]
混合学习域，训练切换压力。
```

流派选择：

```text
选择流派

[稳修] 稳定推进，容错更高
[突击] 连对收益高，答错代价更高
[复盘] 心魔收益高，新题推进较慢
```

每张卡必须包含名称、效果、风险或适用场景。

## 5. 题阵界面

```text
净魔题阵 · 第 3 / 5 题
目标：净化审题失误心魔

题干：
……

A. ……
B. ……
C. ……
D. ……

破招
[稳破] [强攻] [观照]

[提交答案]
```

层级：

1. 题阵进度和目标。
2. 题干。
3. 选项。
4. 破招。
5. 提交按钮。

## 6. 移动端规则

- 选项和提交按钮必须在同一答题区域内。
- 选择答案后，不应滚动到页面底部才能提交。
- 提交按钮在选项后立即出现。
- 题眼短课和辅助说明排在提交按钮之后。
- 长题干可以折叠上方材料，但不能隐藏作答选项。
- 按钮文本在 320px 宽度内不溢出。

## 7. 多选题规则

多选题必须明确显示已选状态和确认动作。

```text
已选：A、C
[提交答案]
```

不要使用点击即提交的多选交互。

## 8. 答后反馈界面

```text
答错

正确答案：B
你的答案：C

题眼短课
考点：……
题眼：……
错因：……

心魔变化
审题失误心魔 +1

[下一题]
```

反馈层级：

1. 正确性。
2. 正确答案和玩家答案。
3. 题眼短课。
4. 心魔变化。
5. 下一题按钮。

## 9. 学习报告界面

```text
学习报告

净魔题阵 · 复盘流派
正确率：3 / 5
目标完成：1 / 2

主要错因
审题失误：限定词漏看

心魔变化
新增：法规主体心魔
净化：审题失误心魔 -1

下一局建议
继续净魔题阵，优先处理法规主体和审题失误的交叉题。

[按建议继续]
[换目标] [查看心魔]
```

学习报告不要求玩家回到复杂入口找下一步。

## 10. 心魔页

心魔页展示可复测的问题。

```text
心魔

审题失误
压力：高
最近表现：3 次漏看限定词
建议：净魔题阵 · 复盘流派

[进入净魔题阵]
```

心魔页不展示内部题库审计字段。

## 11. 状态与符号

可使用简洁符号辅助状态：

| 状态 | 符号 |
|---|---|
| 正确 | ✓ |
| 错误 | ✗ |
| 推荐 | ◆ |
| 心魔 | ※ |
| 下一步 | → |

符号只辅助阅读，不能替代文字解释。

## 12. 禁止项

- 首屏放大量入口。
- 局内显示与本题无关的长期面板。
- 答题按钮远离选项。
- 长解析默认展开并挤压下一题按钮。
- 用内部字段解释玩家状态。
- 使用重图像承载核心反馈。
