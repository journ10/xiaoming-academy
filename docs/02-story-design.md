# 小明书院：文字巡游 — 剧情设计文档

> 文档定位：剧情系统的唯一来源。本游戏为纯文字驱动，所有剧情对话硬编码在 JavaScript 中，由 CSS 渲染为文字界面。无剧本 JSON、无解析器、无图片、无 CG。

---

## 1. 剧情数据来源

**纯文字硬编码：** 剧情数据直接写在代码中，不通过外部文件加载。

| 位置 | 代码 | 内容 |
|---|---|---|
| `core.js` | `chapterNarrativeBeats` | 六章剧情台词（章节开场/通关） |
| `core.js` | `dialogueLine()` | 将角色ID和文本包装为对话对象 |
| `core.js` | `getChapterNarrativeLines()` | 按章节ID读取对应台词 |
| `app.js` | `getIntroDialogue()` | 序章四行台词（硬编码） |
| `app.js` | `getDialogueForChapter()` | 调用 `core.js` 生成章节对话 |

## 3. 剧情渲染方式

剧情由 `app.js` 的 `renderStoryStage()` 函数渲染：

```
┌─────────────────────────────────────────┐
│  kicker: 序章 / 剧情                     │
│  标题: 当前说话者名称                     │
│  引言: 当前台词                          │
│                                          │
│  ┌───────────┐                          │
│  │ 已读记录  │ ← 所有已读对话的文本列表   │
│  └───────────┘                          │
│                                          │
│  [继续] [去练功] [去战斗]                │
│  日志: 当前状态提示                      │
└─────────────────────────────────────────┘
```

**渲染逻辑：**
1. 调用 `getDialogueForChapter(chapter, player)` 生成对话数组
2. 按 `storyIndex` 逐行显示当前台词
3. 已读对话在"已读记录"面板中滚动显示
4. 点击"继续"推进 `storyIndex`，最后一句显示"完成剧情"
5. 点击"去练功"或"去战斗"可跳过剧情直接进入对应场景

## 6. 前端渲染对接

```javascript
// app.js 中的剧情渲染逻辑
function renderStoryStage() {
  const line = storyLines[storyIndex];        // 当前台词
  dom.stage.replaceChildren(textScreen({
    kicker: "序章" / "剧情",
    title: line.speakerName,                   // 说话者名称
    intro: line.text,                           // 当前台词
    body: [
      panel("已读记录", 所有已读台词的文本列表)
    ],
    choices: [
      [storyIndex < 末尾 ? "继续" : "完成剧情", advanceStory],
      ["去练功", () => leaveStoryFor("training")],
      ["去战斗", () => leaveStoryFor("battle")]
    ]
  }));
}
```

## 7. 剧情状态管理

| 状态字段 | 类型 | 说明 |
|---|---|---|
| `player.seenIntro` | boolean | 是否看过序章 |
| `player.storyFlags[chapterId]` | boolean | 是否看过该章剧情 |
| `storyMode` | string | 当前剧情模式："intro" / "intro-forced" / "chapter" |
| `storyIndex` | number | 当前台词索引 |
| `storyLines` | array | 当前对话数组 |
