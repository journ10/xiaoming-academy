## 2. 知识图谱构建

### 2.1 概念节点

每道题归属一个**概念节点**。概念节点是知识图谱的基本单元。

**概念节点命名规则：**
- 格式：`主题 · 一级概念 · 二级概念`
- 示例：`教育法规 · 义务教育法 · 免试入学`

**概念节点数量预估：**

| 主题 | 预估概念节点数 | 说明 |
|---|---|---|
| 教育法规 | 80-100 | 法律条文、权利义务、程序 |
| 教育心理学 | 120-150 | 理论、概念、效应、规律 |
| 教学设计 | 100-130 | 目标、学情、活动、评价 |
| 教师职业道德 | 20-30 | 规范、情境、原则 |
| 班级管理 | 60-80 | 策略、顺序、沟通、突发事件 |
| 儿童发展 | 40-60 | 阶段、理论、年龄节点 |
| 综合知识 | 200+ | 跨主题组合 |

**总计：约 620-750 个概念节点，4374 道当前可玩题映射到这些节点；4680 个源题位作为题源上限另行追踪。**

### 2.2 依赖关系

概念节点之间有三种关系：

| 关系 | 说明 | 示例 |
|---|---|---|
| **前置** | 必须先理解 A，才能理解 B | 免试入学 → 就近入学 → 政府保障 |
| **并列** | A 和 B 是同一层级的概念 | 教师权利 vs 教师义务 |
| **上位** | A 是 B 的上位概念 | 教育法规 → 义务教育法 |

**依赖关系构建方式：**
1. 基于解析文本中的关键词提取
2. 基于教育学知识的人工标注（优先级：高）
3. 基于题目共现的自动推断（如果两道题经常在同一试卷中出现，可能有概念关联）

### 2.3 题目映射到概念节点

```json
{
  "id": "pdf-01241",
  "concept": "教育法规 · 义务教育法 · 政府保障",
  "dependencies": [
    "教育法规 · 义务教育法 · 免试入学",
    "教育法规 · 义务教育法 · 就近入学"
  ],
  "topic": "教育法规",
  "year": "2013",
  "type": "单项选择题",
  // ... 其他字段
}
```

**映射规则：**
- 每道题必须映射到至少一个概念节点
- 多选题/案例分析题可能映射到多个概念节点（跨概念）
- 综合知识题映射到多个主题的概念节点

## 3. 错误模式标注

### 3.1 自动推断错误模式

基于题目特征和选项设计，自动推断可能的错误模式：

| 题目特征 | 推断错误模式 | 标注方式 |
|---|---|---|
| 选项中有相似概念 | 概念混淆 | `errorPattern: "concept-confusion"` |
| 题干含否定词（"不""除外"） | 审题失误 | `errorPattern: "reading-mistake"` |
| 解析中强调"易错点" | 概念混淆 | `errorPattern: "concept-confusion"` |
| 解析中强调"注意" | 审题失误 | `errorPattern: "reading-mistake"` |
| 概念首次出现，无前置题 | 记忆盲区 | `errorPattern: "memory-gap"` |
| 案例分析，情境复杂 | 应用失误 | `errorPattern: "application-error"` |

### 3.2 人工标注规则

对于高优先级题目（高频题、经典题），人工标注错误模式：

```json
{
  "id": "pdf-01241",
  "errorPatterns": [
    {
      "type": "concept-confusion",
      "confusedWith": ["就近入学"],
      "explanation": "容易把'政府保障'和'就近入学'搞混。前者是'提供条件'，后者是'保障入学'。"
    }
  ]
}
```

### 3.3 错误模式与心魔映射

| 错误模式 | 心魔原型 | 净化策略 | 对应讲解 |
|---|---|---|---|
| concept-confusion | 镜像心魔 | 对比表格 | 题眼：对比表格 |
| reading-mistake | 迷雾心魔 | 高亮关键词 | 题眼：关键词高亮 |
| memory-gap | 空洞心魔 | 强制练功 | 题眼：基础概念 |
| application-error | 变形心魔 | 情境匹配 | 题眼：情境分析 |

## 4. 题库格式

### 4.1 标准字段

```json
{
  "id": "pdf-0001",
  "year": "2013",
  "type": "单项选择题",
  "topic": "教育法规",
  "concept": "教育法规 · 义务教育法 · 政府保障",
  "dependencies": ["教育法规 · 义务教育法 · 免试入学"],
  "errorPatterns": ["concept-confusion"],
  "stem": "题干文本",
  "options": [
    { "key": "A", "text": "选项 A" },
    { "key": "B", "text": "选项 B" },
    { "key": "C", "text": "选项 C" },
    { "key": "D", "text": "选项 D" }
  ],
  "answer": "B",
  "explanation": "讲解文本",
  "sourceRef": "来源",
  "difficulty": 1,
  "lesson": {
    "title": "教育法规 · 第 1 题解析",
    "keyPoint": "政府保障",
    "studyPrompt": "练功目标：记住'政府保障'和'就近入学'的区别。"
  },
  "ocr": {
    "requiresReview": false,
    "examTitle": "2013年1月12日深圳事业单位教师招聘考试"
  }
}
```

### 4.2 新增字段（设计版）

| 字段 | 类型 | 说明 | 来源 |
|---|---|---|---|
| `concept` | string | 概念节点路径 | 人工标注 + 自动推断 |
| `dependencies` | array | 前置概念节点 | 知识图谱构建 |
| `errorPatterns` | array | 可能的错误模式 | 自动推断 + 人工标注 |
| `chapterMechanic` | string | 本章特殊机制参数 | 硬编码 |

### 4.3 导入格式

支持两种格式：

```json
// 格式A：题目数组
[
  { /* 题目对象 */ }
]

// 格式B：包含 questions 的对象
{
  "questions": [
    { /* 题目对象 */ }
  ]
}
```

## 5. 代码加载流程

```
app.js 启动
    ↓ fetch("./data/questions.from-pdf.json")
    ↓ parseQuestionImport(json)
    ↓ prepareQuestions(rawQuestions)
    ↓ 自动推断：enemy/realm/heartMethod/difficulty/lesson/concept/dependencies/errorPatterns
    ↓ createStoryChapters(questions)  // 按 topic 分组生成 7 章
    ↓ buildKnowledgeGraph(questions)  // 构建知识图谱
    ↓ 渲染到页面
```
