# 小明书院：内容与题库规范

## 1. 内容定位

题库是游戏的核心资产。运行时必须以历年真题为可信来源，保留原题干、原选项、原答案和解析来源。

游戏层可以增加目标、流派、破招、题眼短课、心魔和学习报告，但不能改变真题本身。

## 4. 题型口径

源题表面题型包括：

- 单项选择题
- 多项选择题
- 判断题
- 案例分析题
- 论述题

运行时按可作答结构处理：

- 单答案题：选择一个选项。
- 多答案题：选择多个选项并确认。
- 二选题：按原选项结构呈现。
- 长题干题：可折叠材料，但不改写题干。
- 论述类内容：如已结构化为选项题，按选项题处理；不进行自由文本自动判分。

## 5. 题目质量规则

题阵推荐优先使用：

- 分类可靠题。
- 质量状态可用题。
- 选项完整题。
- 答案明确题。

不自动推荐：

- 待人工归类题。
- 选项 OCR 明显残缺题。
- 答案或解析无法确认题。
- 需要人工核对来源的题。

这些过滤规则是内容质量规则，不作为玩家主界面信息展示。

## 6. 字段使用

核心字段：

| 字段 | 用途 |
|---|---|
| `stem` | 题干展示 |
| `options` | 原选项展示 |
| `answer` | 判分 |
| `explanation` | 题眼短课和完整解析来源 |
| `difficulty` | 题阵编排 |
| `type` | 作答结构辅助判断 |
| `primaryDomain` / runtime domain | 学习域、报告、推荐 |
| `qualityStatus` | 推荐过滤 |
| `errorPatterns` | 心魔推断 |

旧兼容字段可以保留，但不得作为玩家可见学习域。

## 7. 题眼短课生成规则

题眼短课必须来自题目和解析。

生成时优先提取：

- 考点关键词。
- 条文主体。
- 否定词或限定词。
- 场景行为。
- 相近概念差异。
- 多选项遗漏风险。

不能凭空添加来源外知识。如解析不清晰，使用保守表述并保留完整解析入口。

## 8. 心魔推断规则

心魔可由以下信息推断：

- 玩家答案与正确答案差异。
- 题干关键词。
- 题型。
- 选项数量。
- 是否多选漏选。
- 解析中的概念和限定。
- 玩家历史错因。

心魔推断不是内容审计结论。它服务学习反馈，可随玩家后续表现修正。
