# 小明书院：秘卷巡游 — 实施路线图

> 当前基准：本路线图围绕 `docs/11-roguelite-transformation-plan.md` 的改造执行，不再推进旧“七章章节差异化 + 独立剧情”方案。

## 1. 改造目标

把当前游戏改成一个玩家打开就知道怎么开始的备考 Roguelite：

- 首屏是开局台，不是地图。
- 主循环是一局 5 题，不是章节路线。
- 剧情是事件反馈，不是独立阅读页。
- 学习域来自重新分类审计，不再使用“综合知识”章。
- 战报给出下一局建议，不让玩家回复杂入口里找方向。

## 3. 当前验收标准

| 验收项 | 标准 |
|---|---|
| 首屏清晰 | 首屏出现“今日推荐”“本局目标”“开始一局” |
| 无旧主线 | 首屏和主导航不出现“正式主线”“章节提示”“完成剧情” |
| 单局可玩 | 推荐局能进入 5 题题阵 |
| 模式可选 | 探索局、净化局、冲刺局均可创建 |
| 流派可选 | 稳修、突击、复盘均可创建 |
| 心魔有效 | 有活跃心魔时推荐净化局 |
| 战报有效 | 结算页出现下一局建议 |
| 审计分层 | HUD 不显示源题位、待归类、复核等内容生产指标 |
| 文档一致 | README 与 docs/01-08 不再描述旧章节游戏为当前目标 |
| 测试通过 | `npm test` 全部通过 |

## 4. 后续增强路线

这些不是当前改造完成的阻塞项。

| 优先级 | 增强项 | 目的 |
|---|---|---|
| P1 | 人工处理 114 道待归类题 | 提高正式题池质量 |
| P1 | 让运行时直接读取 primaryDomain | 减少对旧 topic 的兼容依赖 |
| P2 | 更细的错因诊断 | 提升心魔反馈可信度 |
| P2 | 模式胜率和流派胜率统计 | 让报告更像学习仪表盘 |
| P3 | 更丰富的遭遇事件 | 增加娱乐性但不打断刷题 |
| P3 | 视觉 polish 和移动端截图回归 | 进一步改善第一屏观感 |

## 5. 不再推进的旧方向

- 七章地图式主线。
- 独立剧情页和剧情通关门槛。
- “综合知识”终章。
- 章节锁定链。
- 底部导航承载主要玩法入口。
- 在 HUD 展示源题位、复核数、待归类数。
