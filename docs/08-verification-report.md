# 小明书院：验证记录与检查清单

日期：2026-06-24

## 1. 当前设计基准

当前产品形态为文字版真题答题游戏：

- 从开局台进入 5 题题阵。
- 真题题干、选项和作答结构保持原貌。
- 流派提供局前学习策略。
- 破招提供单题轻策略。
- 答后题眼短课解释考点、题眼和错因。
- 错题生成或强化心魔。
- 学习报告给出下一局建议。

## 2. 本轮文档清理目标

- 删除旧改造方案和历史方案文档。
- 将主设计文档统一为“真题题阵”形态。
- 去除重美术、长叙事和过期阶段计划描述。
- 保留内容管线和人工复核资料。
- 保留项目测试命令和验收口径。

## 3. 需要保持一致的玩家词汇

- 开局台
- 题阵
- 流派
- 破招
- 题眼短课
- 心魔
- 学习报告

## 4. 设计一致性检查

检查文档是否仍然描述过期方案、旧入口或阶段性范围措辞。主设计文档不应继续描述过期产品方向。

## 5. 运行验证命令

文档变更不要求启动服务。若后续修改运行时代码，按项目要求运行：

```bash
npm test
```

视觉和移动端交互变更需要额外用 `npm start` 在 4190 端口预览，并在结束前停止服务。

## 6. 手动验收问题

- 玩家打开后是否知道点哪里进入题阵？
- 题干和选项是否始终保持真题原貌？
- 选项作答是否比辅助说明更突出？
- 使用破招后是否仍需要玩家自己作答？
- 答错后是否能看懂为什么错？
- 学习报告是否给出下一局建议？
- 心魔是否来自真实错因，而不是装饰性状态？
