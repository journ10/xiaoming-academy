# 小明书院：秘卷巡游 — 视觉素材与文字美术规范

> 当前基准：运行时以 CSS、排版、Unicode 状态符号和文字氛围构成视觉表现。不再围绕世界地图、章节封印或剧情 CG 组织素材。

## 1. 视觉目标

| 目标 | 说明 |
|---|---|
| 清晰 | 玩家能快速找到“开始一局”和本局目标 |
| 有游戏感 | 题阵、心魔、战报要像局内事件，不像表单 |
| 不喧宾夺主 | 视觉装饰不能遮挡题干、选项和反馈 |
| 可维护 | 无需依赖外部图片生成流程即可运行 |

## 2. 核心视觉语言

| 元素 | 表现 |
|---|---|
| 开局台 | 大标题 + 目标面板 + 主按钮 |
| 模式卡 | 边框、状态标签、推荐高亮 |
| 流派卡 | 名称、收益、风险三层排版 |
| 题阵 | 暗色舞台、题干高对比、选项稳定尺寸 |
| 心魔 | 压力值、错因标签、红色状态 |
| 战报 | 数据摘要 + 下一局建议 |

## 3. Unicode 状态符号

| 含义 | 字符 | 用途 |
|---|---|---|
| 正确 | ✓ | 答对、完成 |
| 错误 | ✗ | 答错、失败 |
| 推荐 | ◆ | 推荐模式或流派 |
| 心魔 | ※ | 错题债、压力 |
| 进度 | █▓░ | 覆盖率、心法 |
| 分隔 | ─ ━ | 信息分区 |
| 方向 | → | 下一步建议 |

避免在主流程中使用会把界面变成旧地图语义的符号，例如地图、锁定章节、封印等。

## 4. 色彩语义

```css
:root {
  --bg-page: #101216;
  --bg-panel: #181b21;
  --bg-panel-soft: #20242c;
  --text-main: #f4efe4;
  --text-muted: #9da3ad;
  --text-line: rgba(255, 255, 255, 0.1);
  --accent: #50d6b1;
  --danger: #e16f7c;
  --gold: #d8b35d;
  --warning: #d49b45;
}
```

| 状态 | 主色 | 用途 |
|---|---|---|
| 推荐/正确 | accent | 选中、答对、目标推进 |
| 心魔/错误 | danger | 答错、压力、风险 |
| 奖励/里程碑 | gold | 战报收益、长期进度 |
| 警告 | warning | 心力不足、复燃风险 |

## 5. 字体排版

| 场景 | 建议 |
|---|---|
| 标题 | 衬线字体，保持书院气质 |
| 数据 | 等宽或清晰无衬线，便于比较 |
| 题干 | 字号大于普通说明，行高充足 |
| 选项 | 固定间距，避免点击目标跳动 |
| 按钮 | 简短动词，不写长句 |

## 6. 动效原则

| 动效 | 触发 | 时长 |
|---|---|---|
| 目标高亮 | 进入开局台 | 200-300ms |
| 答对闪光 | 答题正确 | 150-250ms |
| 答错震动 | 答题错误 | 150-250ms |
| 心魔压力变化 | 心魔生成/净化 | 250-400ms |
| 战报计数 | 进入战报 | 300-500ms |

必须支持 `prefers-reduced-motion`。

## 7. 禁止项

- 运行时依赖章节地图图片。
- 用剧情 CG 承担核心反馈。
- 用大面积装饰图遮挡题干。
- 首屏展示素材墙或角色立绘。
- 把审计指标设计成 HUD 装饰。

## 8. 历史素材

`assets/generated/` 和 `assets/reference/legacy/` 中的历史图片可作为参考或归档，但当前 Roguelite 主流程不依赖它们。后续如果重新引入图片，必须服务开局、局内遭遇、心魔反馈或战报，而不是恢复旧章节地图。
