# 小明书院：秘卷巡游 RPG 重构方案

## Direction Lock

本次不再沿用旧三栏布局。新版本是一个浏览器 RPG：首屏是全屏学院世界地图，玩家在六个章节节点上巡游，触发角色对话、短课练功、题阵战斗和章节封印。学习内容仍来自题库讲解，但在体验上先表现为“剧情任务和战斗准备”，再表现为答题。

## Reference Decisions

- BrowserQuest 提供“网页也可以先像游戏地图”的方向：首屏必须是地图、角色和可点击关卡，而不是题卡工具。
- Narrat 提供“对话、任务、技能检查共存”的方向：讲解与章节目标要由角色对话和任务板承载。
- Phaser/Excalibur 等网页游戏框架强调场景、视觉反馈、动画和状态机；本项目仍保持静态无构建，但借鉴场景制结构。
- Duolingo 的短课、连续练习、能量和 XP 启发成长曲线，但奖励不能鼓励绕过学习；错题不给成长，只给复训目标。

## Game Promise

玩家不是在刷题，而是在书院地图里带队修复真题秘卷。每个章节都是一段剧情任务：角色先说明冲突，玩家通过短课掌握题眼，再用战斗检验。胜利点亮章节星印，失败留下心魔支线。

## Screen Structure

1. `World`：全屏地图背景，六章节点、主线任务、角色立绘、底部对话框。
2. `Story`：视觉小说式对话，角色头像和立绘发言，推动章节目标。
3. `Training`：短课练功，像游戏技能学习；展示题眼、关键概念、记忆卡和“准备完成”反馈。
4. `Battle`：RPG 战斗场景，角色立绘、心魔敌人、技能按钮、答案符文、特效结算。
5. `Roster`：队伍与成长，展示等级、星辉、羁绊、章节星印、心法树。
6. `Review`：心魔支线，错题以敌人列表出现，净化后获得小墨羁绊与星印修复度。

## Narrative Spine

- 明澈：书院导师，序章说明“秘卷失光”的主线。
- 阿芷：练功伙伴，负责把题库讲解转为短课与题眼卡。
- 青岚：战斗伙伴，负责破招、连击、强反馈。
- 小墨：秘卷灵，负责心魔支线、章节封印和终局归元。

六章结构：
1. 律令花窗：教育法规，学会法条关键词。
2. 观心花园：教育心理学，学会学习与动机题眼。
3. 授业工坊：教学设计，学会目标、学情、活动、评价。
4. 师德星廊：教师职业道德，学会情境边界。
5. 统御回廊：班级管理，学会事件处理顺序。
6. 童心星谷：儿童发展，学会发展特点与支持策略。

## Learning Loop

- 每个章节节点显示“剧情目标 + 未掌握题数 + 推荐行动”。
- 第一次进入题目不是直接答题，而是触发阿芷短课。
- 短课只展示必要题眼、来源和 1 个记忆动作，不做长篇教材页面。
- 短课完成获得星辉、修为、阿芷羁绊和战斗准备状态。
- 战斗答对获得伤害、灵页、心法、青岚羁绊、连击和章节星印进度。
- 战斗答错不获得成长，生成心魔和复训任务。
- 心魔净化需要复看讲解并连续答对，净化后获得小墨羁绊。

## Growth Curve

- `playerLevel`：由修为 XP 推导，每级提升可见称号和最大能量。
- `energy`：替代旧心力，进入战斗消耗，答对连胜返还；错题消耗但不直接阻断首轮体验。
- `starGlimmer`：学习和答对获得，用于视觉强化和队伍羁绊展示。
- `bond`：角色羁绊分工明确，学习、战斗、净化分别提升不同角色。
- `chapterSeal`：章节通关必须满足全部题目已学习、已答对、无活跃心魔、主题心法达到门槛。
- `streak`：连续正确只放大视觉和小额能量返还，不替代题库掌握。

## Visual Direction

使用生成位图作为主资产，不再用 SVG 角色妥协。世界地图、战斗背景、头像、立绘都使用 `assets/generated/`。UI 是“RPG HUD + 视觉小说对话框 + 地图节点”，按钮和题卡只是战斗中的局部控件。

## Implementation Shape

- `core.js` 保留题目解析、成长、章节、战斗和心魔纯逻辑。
- `app.js` 改成 scene controller：`world`, `story`, `training`, `battle`, `roster`, `review`, `report`。
- `index.html` 改为一个 RPG 舞台容器，不再按左中右三栏布局。
- `styles.css` 以全屏背景、角色立绘、底部对话框、HUD、动画特效为主。
- `tests/core.test.mjs` 增加能量、场景流、剧情步骤、成长曲线和章节通关测试。

## Acceptance Criteria

- 第一屏必须像网页 RPG：地图、章节节点、角色立绘、HUD、对话，而不是题库后台。
- 玩家第一次运行能通过序章理解“练功再战斗”的玩法。
- 角色对话推动章节，不是静态说明文字。
- 学习短课像技能准备，答题像战斗检验。
- 答对和学习给强正反馈；答错不给成长，只给心魔支线。
- 全章节通关代表当前题库完成熟练覆盖。
- 桌面和移动端都无横向滚动，关键按钮可触达。
