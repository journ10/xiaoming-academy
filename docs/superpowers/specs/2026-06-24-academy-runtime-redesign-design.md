# 小明书院运行时重构设计

日期：2026-06-24

## 背景

本次改造以新版 `academy-design.pen` 和现有设计文档为准，彻底重构浏览器运行时。旧版本中的章节、地图、手账、日课、材料、心力、仪表盘、独立短课页等内容不再兼容保留。题库资产、题目标准化、答案判定、可靠题筛选等可继续复用，但用户可见体验按新版五入口结构重建。

主体验保留：

- 开局台
- 题阵
- 心魔
- 学习报告
- 设置

## 目标

1. 将产品体验收缩为文字优先的 5 题题阵学习循环。
2. 完整实现明亮 / 夜读双主题和设置页切换。
3. 移除旧体验入口、旧状态、旧文案和旧系统噪声。
4. 保留真题题干、选项、答案结构和题库可靠性边界。
5. 让心魔、题眼短课、学习报告都服务下一局学习决策。

## 非目标

- 不兼容旧版本存档。
- 不恢复章节地图、视觉 RPG、手账、日课、材料、心力、仪表盘。
- 不把待人工归类题放入推荐题阵。
- 不重做 PDF 抽取和题库生成管线。
- 不让 PC 题号进度圆点支持跳题。

## 信息架构

### 开局台

开局台只解决“现在进入哪一局题阵”。

无未完成题阵时，系统根据当前存档动态生成推荐。没有历史数据时默认：

- 题阵目标：拓新题阵
- 推荐流派：稳修
- 目标：完成 5 道新题并新增题眼

有未完成题阵时，开局台优先展示继续当前题阵。

`换目标` 在开局台卡片内切换：

- 拓新题阵
- 净魔题阵
- 冲刺题阵

流派在开局台原地选择：

- 稳修
- 突击
- 复盘

选择流派不会立即开始，用户点击 `进入题阵` 后才进入题阵。

### 题阵

题阵固定 5 题。题阵目标决定选题倾向：

- 拓新题阵：优先可靠新题。
- 净魔题阵：优先当前高压心魔相关题和历史错题。
- 冲刺题阵：优先多学习域混合题。

如果当前目标下可用题不足 5 道，从可靠历史题中补足，优先易错题。

每题显示：

- 当前进度
- 题阵目标
- 题干
- 原选项
- 破招选项
- 已选答案
- 提交按钮

破招包括：

- 稳破
- 强攻
- 观照

观照一旦触发，只显示一句题眼提示，不展示完整答案或解析。本题锁定为观照收益，不能切回稳破或强攻。玩家仍必须自己选择选项并提交。

离开题阵再返回时，保留当前题序、已选答案、破招、观照提示状态和已结算题状态。

### 答后反馈

提交后进入本题反馈状态，包含：

- 答对 / 答错状态
- 正确答案和玩家答案
- 题眼短课
- 心魔变化
- 下一题按钮

题眼短课是答后反馈的一块内容，不再有独立页面。短课默认只展示考点、题眼、错因的短说明。

### 心魔

心魔页展示当前最需要复测的问题。

内容包括：

- 当前高压心魔
- 压力
- 最近表现
- 建议
- 待复测心魔列表
- `进入净魔题阵`

从心魔页进入净魔题阵时，只预选或切换到净魔目标，不默认流派；流派仍由开局台选项确认。抽题优先当前高压心魔相关题。

### 学习报告

学习报告只保留设计稿里的摘要结构：

- 本局摘要
- 本局收获
- 下一步建议
- 再来一局

旧事件列表、材料、手账收藏、复习清单导出等内容全部移除出主界面。

### 设置

设置页保留：

- 存档码导出
- 存档码导入
- 重置
- 明亮 / 夜读主题切换

导入存档码会覆盖当前新版进度。旧版存档不保证可用。

## 运行时数据模型

新版存档使用专用结构，旧字段不迁移。

```js
{
  version: 2,
  theme: "light" | "night",
  answered: {
    [questionId]: {
      attempts: number,
      correct: number,
      wrong: number,
      lastAnswer: string,
      lastCorrect: boolean,
      lastAt: string
    }
  },
  demons: {
    [demonId]: {
      id: string,
      type: string,
      pressure: number,
      questionIds: string[],
      recentText: string,
      purified: boolean,
      lastAt: string
    }
  },
  currentRun: {
    targetId: "explore" | "purify" | "sprint",
    styleId: "steady" | "assault" | "review",
    questionIds: string[],
    currentIndex: number,
    answers: {
      [questionId]: {
        selectedKeys: string[],
        breakMoveId: "steady" | "assault" | "observe",
        observeRevealed: boolean,
        submitted: boolean,
        isCorrect: boolean,
        correctAnswer: string
      }
    }
  } | null,
  lastReport: object | null
}
```

## Core 模块设计

`core.js` 重建为纯玩法核心，导出稳定的函数：

- `prepareQuestionBank(rawQuestions)`
- `selectRunQuestions(questions, state, options)`
- `createRun(questions, state, options)`
- `normalizeAnswer(answer, options)`
- `judgeAnswer(question, selectedKeys)`
- `createObservationHint(question)`
- `applyAnswer(state, run, question, selectedKeys, breakMoveId)`
- `createDemonUpdate(state, question, result)`
- `createLearningReport(state, run)`
- `createStartRecommendation(questions, state)`

保留题库完整性规则：

- 使用内置 PDF 题库作为可玩题来源。
- 使用六个真实学习域。
- 排除待人工归类和质量不稳题。
- 不恢复 `综合知识` 作为可玩兜底域。
- 保留题干、选项、答案结构。

## App 模块设计

`app.js` 只管理新版 UI 状态机：

- `start`
- `run`
- `demons`
- `report`
- `settings`

主要职责：

- 加载题库和题目分片。
- 加载、保存、导入、导出新版存档。
- 渲染五个主场景。
- 管理开局台目标/流派选择。
- 管理当前题阵进度和题内交互。
- 管理主题切换。

## 样式设计

`styles.css` 按设计稿重建，不保留旧 RPG/手账样式。

基础变量：

- 明亮主题：浅灰背景、白色卡片、蓝色主操作、紫色观照提示、绿色正确反馈、橙色心魔压力。
- 夜读主题：深色背景、深蓝卡片、蓝色主操作、紫色观照提示、绿色正确反馈、橙色心魔压力。

布局：

- 移动端：390px 设计稿优先，底部固定五入口导航。
- PC 端：顶部导航，题阵页显示左侧 1-5 题进度栏。
- 固定导航和底部操作不得遮挡题干、选项、提交按钮或反馈内容。

## 测试设计

测试按新版口径同步重写或更新。

必须覆盖：

- 玩家可见文案不出现旧词：地图、练功、战斗、学习风格、章节机制、章节封印、题眼手账、今日清单、学习仪表盘、材料、心力。
- 开局台默认推荐为拓新题阵 + 稳修。
- 未完成题阵优先继续。
- 换目标在拓新题阵、净魔题阵、冲刺题阵之间切换。
- 流派选择不立即开始题阵。
- 题阵固定 5 题，不足时补足可靠题。
- 多选答案规范化判定。
- 观照只显示提示，不显示完整答案，并锁定观照收益。
- 答错生成或强化心魔。
- 心魔页进入净魔题阵优先抽当前高压心魔相关题，但不默认流派。
- 学习报告只显示摘要、收获和下一步。
- 设置页主题切换和存档码导入导出可用。
- 旧玩家可见词不回流到 `index.html`、`app.js`、`core.js`。

运行时改造完成前，必须运行：

```bash
npm test
```

视觉改造完成后，使用：

```bash
npm start
```

固定在 4190 端口预览，结束前停止服务。

## 实施顺序

1. 重写 `core.js` 的新版数据模型、抽题、判分、心魔和报告。
2. 重写 `app.js` 的五场景状态机和存档码机制。
3. 重写 `styles.css`，实现明亮 / 夜读主题和响应式布局。
4. 更新 `index.html` 的标题、资源版本和外壳结构。
5. 更新测试到新版口径。
6. 运行 `npm test`。
7. 启动 4190 端口做浏览器视觉检查，并停止服务。

## 已确认决策

- 彻底重构，旧内容可以删除。
- 主入口只保留开局台、题阵、心魔、学习报告、设置。
- 完整实现明亮 / 夜读双主题。
- 多余旧入口和旧文案直接移除。
- 开局台流派选项原地选择，进入题阵按钮才开始。
- 观照触发后锁定本题观照收益。
- 心魔页进入净魔题阵不默认流派。
- 旧存档不用兼容。
- 题阵不足 5 题时用可靠历史题补足，优先易错题。
