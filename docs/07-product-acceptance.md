# 小明书院：产品验收口径

## 1. 体验验收

- 玩家进入后 10 秒内能理解推荐题阵和主按钮。
- 一局题阵固定 5 题，并显示当前进度。
- 每局有明确目标，学习报告能结算目标完成情况。
- 玩家完成一局后能直接按建议进入下一局。
- 心魔能成为下一局推荐理由。

## 2. 真题可信度验收

- 题干与题库原文一致。
- 选项与题库原文一致。
- 作答结构与题库答案结构一致。
- 破招不替玩家作答。
- 题眼短课不改变题意。
- 学习报告基于真实作答结果。

## 3. 学习验收

- 答错后必须出现正确答案和题眼短课。
- 答错后必须生成或更新心魔。
- 心魔净化必须依赖再次作答正确。
- 学习报告必须指出主要错因。
- 下一局建议必须来自错因、覆盖度或稳定度。

## 4. 文字反馈验收

- 开局台文案说明推荐理由和本局目标。
- 题眼短课默认 1 到 3 句。
- 答错反馈描述错因，不羞辱玩家。
- 报告文案给出可执行下一步。
- 不用长段文字解释基本操作。

## 5. UI 验收

- 题干和选项是局内最高优先级内容。
- 选择答案后提交按钮可立即到达。
- 移动端不需要滚动到底才能确认答案。
- 多选题能清楚显示已选项。
- 答后反馈中的下一题按钮清晰可见。
- 学习报告的继续按钮优先级最高。

## 6. 内容验收

- 待人工归类题不进入系统推荐题阵。
- 质量不稳题不进入默认推荐。
- 六个学习域用于推荐和报告。
- 题库审计字段不出现在玩家主界面。
- 题眼短课可以追溯到题目或解析。

## 7. 自动化检查建议

运行时代码变更后按项目要求运行：

```bash
npm test
```

文档变更后应搜索过期方案标题、旧入口词和阶段性范围措辞；主设计文档不应继续描述过期产品方向。
