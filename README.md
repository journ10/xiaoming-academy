# 小明书院：秘卷巡游

一个无后端、无外部依赖的网页 RPG。当前版本已经从三栏答题工具破坏性重构为全屏 RPG 舞台：生成位图地图、角色头像、人物立绘、视觉小说式对话、章节主线、短课练功、题阵战斗、心魔支线、星辉、能量、羁绊、等级和章节星印。

## 运行

```bash
cd /Users/wangzexu/Documents/Codex/academy
npm run start
```

```text
http://localhost:4173
```

## 当前玩法

- 第一屏是学院世界地图，不再是题库布局。六个章节节点直接显示在地图上。
- 首次进入触发序章对话，人物头像和立绘会推动剧情说明。
- 每章先看主线剧情，再进入阿芷短课练功，最后由青岚带入题阵战斗。
- 短课像 RPG 技能准备，只展示题眼、来源和记忆动作。
- 战斗在生成的战斗背景上进行，玩家选择稳破、强攻或观照，再选择答案符文。
- 右侧任务板显示剧情、练功、检验、心魔和心法门槛。
- 答对题目会提升星辉、修为、灵页、心法和青岚羁绊；答错不会给成长奖励，只会生成或强化心魔。
- 章节通关要求本章剧情已看、题目全部练过、答对过、没有活跃心魔，并达到心法门槛。
- 全部章节通关代表当前题库完成熟练覆盖。
- 每局 5 题，完成或失败后生成破阵战报。

## 美术资源

生成位图资源保存在 `assets/generated/`，按功能分组：

- `backgrounds/`：世界地图、战斗背景、章节背景和场景背景
- `characters/`：四名角色头像、立绘、cutout、角色变体和角色资产表
- `demons/`、`items/`、`artifacts/`、`nodes/`、`seals/`：运行时内容图标和立绘
- `ui/`、`map/`、`story/`、`training/`、`battle/`、`mind-demon/`、`report/`、`daily/`、`roster/`、`artifact-ui/`、`fx/`：按页面和功能拆分的 UI/状态/特效素材
- `brand/`：书院 logo 和通用墨迹标记

历史 SVG 和视觉方向参考图已归档在 `assets/reference/legacy/`，不再作为运行时主素材。

## 内容与 PDF

`docs/10小明课堂 《历年真题答案》.pdf` 是无文本层扫描版答案解析页，主要包含参考答案与解析，不包含完整原题选项。

当前已生成两类 PDF 接入产物：

- `data/pdf-ocr-pages.jsonl`：512 页逐页 OCR 原文，保留行文本、行框和置信度。
- `data/questions.from-pdf.json`：按项目题库格式生成的“答案回忆/解析题”，可通过页面 JSON 导入入口导入。

`data/pdf-ocr-sample.json` 只用于保留早期 PDF 内容接入样本，不作为当前主流程题库。

由于源 PDF 不是完整真题卷，`data/questions.from-pdf.json` 不会补造原题干和原选项；其中 `ocr.requiresReview=true` 的条目需要按原页图人工复核后再作为正式题库使用。

内容接入流程见 `docs/content-pipeline.md`。

## 题库格式

页面仍保留 JSON 导入入口。格式支持：

```json
{
  "questions": []
}
```

或直接使用题目数组：

```json
[]
```

单题字段：

```json
{
  "id": "question-001",
  "year": "2026",
  "type": "单项选择",
  "topic": "教育法规",
  "stem": "题干文本",
  "options": [
    { "key": "A", "text": "选项 A" },
    { "key": "B", "text": "选项 B" }
  ],
  "answer": "B",
  "explanation": "讲解文本",
  "sourceRef": "讲解来源",
  "difficulty": 2
}
```

多选题的 `answer` 写连续字母，例如 `ABC`。
